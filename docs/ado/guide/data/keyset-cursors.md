# <a name="keyset-cursors"></a>키 집합 커서
키 집합 커서는 정적 및 동적 커서 간에 기능 변경 내용을 검색 하는 기능 제공 합니다. 정적 커서처럼 항상 결과 집합의 멤버 자격과 순서에 대한 변경 내용을 검색하지는 못합니다. 동적 커서처럼 결과 집합의 행 값에 대한 변경 내용을 검색합니다.  
  
 키 집합 커서는 키 집합이라는 고유 식별자(키) 집합으로 제어됩니다. 키는 결과 집합에서 행을 고유하게 식별하는 열 집합으로 작성됩니다. 키 집합은 쿼리 문에서 반환된 모든 행의 키 값 집합입니다.  
  
 키 집합 커서를 사용하여 커서의 각 행에 대해 키를 빌드 및 저장하고 클라이언트 워크스테이션 또는 서버에 저장합니다. 각 행에 액세스하면 저장된 키를 사용하여 데이터 원본에서 현재 데이터 값을 페치합니다. 키 집합 커서에서 키 집합이 완전히 채워지면 결과 집합 멤버 자격이 고정됩니다. 이후 멤버 자격에 영향을 주는 추가 또는 업데이트는 다시 열 때까지 결과 집합의 일부가 아닙니다.  
  
 결과 집합을 스크롤할 (키 집합 소유자 또는 다른 프로세스에 의해 생성) 하는 데이터 값의 변경 내용이 표시 됩니다. 커서 외부에서 수행한 삽입은(다른 프로세스에 의해)은 커서를 닫았다가 다시 열 때만 볼 수 있습니다. 커서 내에서 수행한 삽입은 결과 집합의 끝에 표시됩니다.  
  
 키 집합 커서는 삭제 된 행을 검색할 때 행은 결과 집합에 "구멍"으로 표시 됩니다. 행의 키는 키 집합에 있지만 행은 결과 집합에 더 이상 존재하지 않습니다. 행에서 키 값이 업데이트 되 면 이러한 행 결과 집합의 구멍으로도 표시 되도록 행 삭제 되 고 그런 다음 삽입으로 간주 됩니다. 키 집합 커서는 항상 검색 하 고 다른 프로세스에 의해 삭제 된 행, 하는 동안 자체를 삭제 하는 행에 대 한 키를 제거 필요에 따라 해당 합니다. 이 작업을 수행 하는 키 집합 커서는 증명 정보 제거 되었으므로 자체에서 삭제를 검색할 수 없습니다.  
  
 키 열에 대 한 업데이트 뒤에 새 키를 삽입 하는 이전 키를 삭제 하는 것과 같습니다. 새 키 값을 업데이트 하지 않는 커서를 통해 하는 경우에 표시 되지 않습니다. 커서를 통해 업데이트 된 경우 새 키 값을 결과 집합의 끝에 표시 됩니다.  
  
 표준 키 집합 커서를 호출 하는 키 집합 커서에 대 한 변형이 있습니다. 키 집합 기반 표준 커서 결과 집합의 행 멤버 자격 및 행의 순서는 커서가 열릴 하지만 커서 소유자가 만든 값이 변경으로 고정 됩니다.에 커밋된 변경 내용을 다른 프로세스에 의해 표시 됩니다. 변경 작업이 멤버 자격에 대 한 행 또는 행의 순서가 결정 하는 경우 행 사라집니다 또는 커서를 닫았다가 않는 한 이동 하지 않습니다. 삽입 된 데이터에 표시 되지 않지만 기존 데이터에 변경 행이 인출 하는 대로 나타나지 합니다.  
  
 키 집합 커서를 올바르게 사용 하려면 데이터 변경 내용에 대 한 민감도 여러 가지 상황에 따라 달라 지므로 위에 설명 된 대로 어렵습니다. 그러나 응용 프로그램에 동시 업데이트로 신경 쓰지 않기, 잘못 된 키를 프로그래밍 방식으로 처리할 수 있습니다 및 특정 키가 지정 된 행에 직접 액세스 해야, 키 집합 커서를 작동 않을 수 있습니다. 사용 하 여는 **adOpenKeyset CursorTypeEnum** ado에서 키 집합 커서를 사용 하도록 지정할 수 있습니다.
