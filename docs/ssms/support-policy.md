# <a name="sql-server-management-studio-ssms-support-policy"></a>SSMS(SQL Server Management Studio) 지원 정책

SQL Server Management Studio에 대한 지원 정책은 다음과 같습니다.

## <a name="support-policy-for-ssms"></a>SSMS에 대한 정책 지원
- SSMS 17.0부터는 SQL 도구 팀이 [Microsoft 최신 수명 주기 정책](https://support.microsoft.com/help/30881/modern-lifecycle-policy)을 채택했습니다.
- 원래 [최신 수명 주기 정책 알림](https://support.microsoft.com/help/447912/announcing-microsoft-modern-lifecycle-policy)을 읽어 보세요.
- 추가 정보는 [최신 정책 FAQ](https://support.microsoft.com/help/30882/modern-lifecycle-policy-faq)를 참조하세요.

## <a name="ssms-updates"></a>SSMS 업데이트 

Microsoft에서는 연간 수 차례 SSMS(SQL Server Management Studio)의 업데이트를 릴리스할 계획입니다. 버전 18.0부터 모든 새 기능뿐만 아니라 모든 보안 업데이트, 중요 업데이트 및 핫픽스가 주 버전의 최신 포인트 버전으로만 릴리스됩니다. SSMS의 새 버전이 시중에 릴리스되면 주 버전 내의 포인트 버전이든 주 버전 자체이든 모든 이전 버전은 [최신 수명 주기 정책](https://support.microsoft.com/help/30881/modern-lifecycle-policy)에 따라 지원되지 않습니다.


예를 들어 SSMS 17.7은 2018년 5월에 릴리스되었습니다. 버전 17.7은 17.8.1이 릴리스된 경우 2018년 6월까지 완벽하게 지원됩니다. 2018년 6월에 버전 17.7은 지원되지 않습니다. 버전 17.9가 2018년 9월에 릴리스된 경우 이전 버전 모두 지원되지 않습니다. 

최상의 환경을 위해 고객은 https://aka.ms/ssms 를 통해 최신 버전을 설치하는 것이 좋습니다.  

**완벽한 지원** 서비스 단계: 현재 분기 최신 버전의 SQL Server Management Studio를 실행하는 경우 보안 업데이트, 중요 업데이트, 새로운 기능 등 모든 업데이트를 받을 수 있습니다.



![지원 매트릭스](./media/ssms-supportpolicy/support-policy.png)


> [!NOTE]
> 현재 분기 최신 버전은 항상 **완벽한 지원** 서비스 단계를 제공합니다. 이 지원 문은 중요 업데이트를 보증하는 코드 오류가 발생한 경우 이를 해결하려면 현재 분기 최신 버전을 설치해야 한다는 의미입니다.
