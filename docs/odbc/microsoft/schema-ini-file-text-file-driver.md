# <a name="schemaini-file-text-file-driver"></a>Schema.ini 파일(텍스트 파일 드라이버)
텍스트 드라이버를 사용하면 스키마 정보 파일을 사용하여 텍스트 파일의 형식이 결정됩니다. 스키마 정보 파일은 항상 Schema.ini라는 이름으로 지정되며 항상 텍스트 데이터 원본과 동일한 디렉터리로 유지됩니다. 스키마 정보 파일은 IISAM에 파일의 일반 형식, 열 이름 및 데이터 형식 정보 및 기타 여러 데이터 특성에 대한 정보를 제공합니다. Schema.ini 파일은 항상 고정 길이 데이터에 액세스하는 데 필요합니다. 텍스트 테이블에 DateTime, 통화 또는 소수점 데이터가 포함되어 있거나 테이블의 데이터 처리를 보다 세밀한 제어하려는 경우 Schema.ini 파일을 사용해야 합니다.  
  
> [!NOTE]  
>  텍스트 ISAM은 Schema.ini가 아닌 레지스트리에서 초기 값을 가져옵니다. 모든 새 텍스트 데이터 테이블에 동일한 기본 파일 형식이 적용됩니다. CREATE TABLE 문으로 만든 모든 파일은 텍스트 **형식 정의** 대화 상자에서 파일 형식 값을 선택하여 \<설정된 것과 동일한 기본 서식 값을 상속하며, 기본값은 **테이블** 목록에서 선택>. 레지스트리의 값이 Schema.ini의 값과 다른 경우 레지스트리의 값은 Schema.ini의 값으로 덮어씁니까?  
  
## <a name="understanding-schemaini-files"></a>스키마.ini 파일 이해  
 Schema.ini 파일은 텍스트 파일의 레코드에 대한 스키마 정보를 제공합니다. 각 Schema.ini 항목은 테이블의 다섯 가지 특성 중 하나를 지정합니다.  
  
-   텍스트 파일 이름  
  
-   파일 형식  
  
-   필드 이름, 너비 및 유형  
  
-   문자 집합  
  
-   특수 데이터 유형 변환  
  
 다음 섹션에서는 이러한 특성에 대해 설명합니다.  
  
## <a name="specifying-the-file-name"></a>파일 이름 지정  
 Schema.ini의 첫 번째 항목은 항상 대괄호로 둘러싸인 텍스트 원본 파일의 이름입니다. 다음 예제에서는 Sample.txt 파일의 항목을 보여 줍니다.  
  
```  
[Sample.txt]  
```  
  
## <a name="specifying-the-file-format"></a>파일 형식 지정  
 Schema.ini의 **형식** 옵션은 텍스트 파일의 형식을 지정합니다. 텍스트 IISAM은 대부분의 문자 구분 파일에서 자동으로 형식을 읽을 수 있습니다. 이중 따옴표(")를 제외한 모든 단일 문자를 파일의 구분 기호로 사용할 수 있습니다. Schema.ini의 **형식** 설정은 파일별로 Windows 레지스트리의 설정을 재정의합니다. 다음 표에는 **Format** 옵션에 대한 유효한 값이 나열되어 있습니다.  
  
|형식 지정자|테이블 형식|스키마.ini 형식 문|  
|----------------------|------------------|---------------------------------|  
|**탭 구분**|파일의 필드는 탭으로 구분됩니다.|형식=탭해제|  
|**CSV 구분**|파일의 필드는 쉼표(쉼표로 구분된 값)로 구분됩니다.|형식=CSVDelimited|  
|**사용자 지정 구분**|파일의 필드는 대화 상자에 입력하도록 선택한 모든 문자로 구분됩니다. 이중 따옴표(")를 제외한 모든 값은 공백을 포함하여 허용됩니다.|형식=구분(사용자*지정 문자)*<br /><br /> 또는<br /><br /> 구분 기호가 지정되지 않은 경우:<br /><br /> 형식=구분()|  
|**고정 길이**|파일의 필드는 고정된 길이입니다.|형식=고정 길이|  
  
## <a name="specifying-the-fields"></a>필드 지정  
 다음과 같은 두 가지 방법으로 문자 구분 텍스트 파일에 필드 이름을 지정할 수 있습니다.  
  
-   테이블의 첫 번째 행에 필드 이름을 포함하고 **ColNameHeader를** **True로 설정합니다.**  
  
-   각 열을 숫자로 지정하고 열 이름 및 데이터 형식을 지정합니다.  
  
 각 열을 숫자로 지정하고 고정 길이 파일의 열 이름, 데이터 형식 및 너비를 지정해야 합니다.  
  
> [!NOTE]  
>  Schema.ini의 **ColNameHeader** 설정은 파일별로 Windows 레지스트리의 **FirstRowHasNames** 설정을 재정의합니다.  
  
 필드의 데이터 형식도 결정할 수 있습니다. **MaxScanRows** 옵션을 사용하여 열 유형을 결정할 때 검색해야 하는 행 수를 나타냅니다. **MaxScanRows를** 0으로 설정하면 전체 파일이 검색됩니다. Schema.ini의 **MaxScanRows** 설정은 파일별로 Windows 레지스트리의 설정을 재정의합니다.  
  
 다음 항목은 Microsoft Jet가 테이블의 첫 번째 행에 있는 데이터를 사용하여 필드 이름을 결정하고 전체 파일을 검사하여 사용되는 데이터 형식을 결정해야 한다는 것을 나타냅니다.  
  
```  
ColNameHeader=True  
MaxScanRows=0  
```  
  
 다음 항목은 문자 구분 파일의 선택 사항이며 고정 길이 파일에 필요한 열**번호(Col**_n)_ 옵션을 사용하여 테이블의 필드를 지정합니다. 이 예제에서는 두 필드에 대한 Schema.ini 항목, 10자 CustomerNumber 텍스트 필드 및 30자 고객 이름 텍스트 필드를 보여 주십니다.  
  
```  
Col1=CustomerNumber Text Width 10  
Col2=CustomerName Text Width 30  
```  
  
 **Col**_n의_ 구문은 다음과 입니다.  
  
```  
  
n=ColumnNametype [#]  
```  
  
## <a name="remarks"></a>설명  
 다음 표는 **Col**_n_ 항목의 각 부분에 대해 설명합니다.  
  
|매개 변수|설명|  
|---------------|-----------------|  
|*ColumnName*|열의 텍스트 이름입니다. 열 이름에 포함된 공백이 포함된 경우 이중 따옴표로 둘러싸야 합니다.|  
|*종류*|데이터 형식은 다음과 같습니다.<br /><br /> **마이크로소프트 제트 데이터 형식**<br /><br /> bit<br /><br /> Byte<br /><br /> Short<br /><br /> long<br /><br /> 통화<br /><br /> Single<br /><br /> Double<br /><br /> DateTime<br /><br /> 텍스트<br /><br /> 메모<br /><br /> **ODBC 데이터 유형** Char(텍스트와 동일)<br /><br /> 플로트(더블과 동일)<br /><br /> 정수(짧은 것과 동일)<br /><br /> 롱차르(메모와 동일)<br /><br /> 날짜 *날짜 형식*|  
|**너비**|리터럴 문자열 `Width`값입니다. 다음 숫자가 열의 너비를 지정했음을 나타냅니다(문자 구분 된 파일의 경우 선택 사항, 고정 길이 파일에 필요).|  
|*#*|열의 너비를 지정하는 정수 **값입니다(너비가** 지정된 경우 필요함).|  
  
## <a name="selecting-a-character-set"></a>문자 집합 선택  
 ANSI와 OEM의 두 문자 집합 중에서 선택할 수 있습니다. Schema.ini의 **CharacterSet** 설정은 파일별로 Windows 레지스트리의 설정을 재정의합니다. 다음 예제에서는 ANSI로 문자 집합을 설정하는 Schema.ini 항목을 보여 주며 있습니다.  
  
```  
CharacterSet=ANSI  
```  
  
## <a name="specifying-data-type-formats-and-conversions"></a>데이터 형식 형식 및 변환 지정  
 Schema.ini 파일에는 데이터를 변환하거나 표시하는 방법을 지정하는 데 사용할 수 있는 몇 가지 옵션이 포함되어 있습니다. 다음 표에는 이러한 각 옵션이 나열되어 있습니다.  
  
|옵션|설명|  
|------------|-----------------|  
|**DateTimeFormat**|날짜와 시간을 나타내는 형식 문자열로 설정할 수 있습니다. 가져오기/내보내기의 모든 날짜/시간 필드가 동일한 형식으로 처리되는 경우 이 항목을 지정해야 합니다. 오전을 제외한 모든 마이크로소프트 제트 형식. 오후 시간이 모두 이 지원됩니다. 형식 문자열이 없는 경우 Windows 제어판 짧은 날짜 그림 및 시간 옵션이 사용됩니다.|  
|**십진수 기호**|정수를 숫자의 소수 부분에서 분리하는 데 사용되는 단일 문자로 설정할 수 있습니다.|  
|**숫자 자릿수**|숫자의 소수 자릿수를 나타냅니다.|  
|**번호 선행 제로**|소수점 값이 1보다 크고 -1보다 작은 소수점 값이 선행 0을 포함해야 하는지 여부를 지정합니다. 이 값은 False(선행 영점 없음) 또는 True일 수 있습니다.|  
|**CurrencySymbol**|텍스트 파일의 통화 값에 사용할 수 있는 통화 기호를 나타냅니다. 예를 들어 달러 기호($) 및 Dm이 있습니다.|  
|**통화포형식**|다음 값 중 한 값으로 설정할 수 있습니다.<br /><br /> - 분리가없는 통화 기호 접두사 ($ 1)<br />- 분리가없는 통화 기호 접미사 (1 $)<br />- 하나의 문자 구분통화 기호 접두사 ($ 1)<br />- 하나의 문자 분리와 통화 기호 접미사 (1 $)|  
|**통화 자릿수**|통화 금액의 소수 부분에 사용되는 숫자 수를 지정합니다.|  
|**통화Neg 형식**|다음 값 중 하나를 사용할 수 있습니다.<br /><br /> - ($1)<br />- -$1<br />- $-1<br />- $1-<br />- (1$)<br />- -1$<br />- 1-$<br />- 1$-<br />- -1 $<br />- -$ 1<br />- 1 $-<br />- $ 1 -<br />- $ 1<br />- 1- $<br />- ($ 1)<br />- (1 $)<br /><br /> 이 예제에서는 달러 기호를 보여 주지만 실제 프로그램에서 적절한 **CurrencySymbol** 값으로 바꿔야 합니다.|  
|**통화천 기호**|텍스트 파일에서 통화 값을 수천 으로 구분하는 데 사용할 수 있는 단일 문자 기호를 나타냅니다.|  
|**통화장식기호**|통화 금액의 소수 부분에서 전체를 구분하는 데 사용되는 단일 문자로 설정할 수 있습니다.|  
  
> [!NOTE]  
>  항목을 생략하면 Windows 제어판의 기본값이 사용됩니다.
