# <a name="oracle-software-patches"></a>Oracle 소프트웨어 패치
> [!IMPORTANT]  
>  이 기능은 Windows의 이후 버전에서 제거 됩니다. 새 개발 작업에서는 이 기능을 사용하지 않도록 하고, 현재 이 기능을 사용하는 애플리케이션은 수정하세요. 대신, Oracle에서 제공 하는 ODBC 드라이버를 사용 합니다.  
  
 여러 Microsoft 제품 및 기술, Microsoft ODBC Driver for Oracle과 Microsoft OLE DB Provider for Oracle, 인터넷 정보를 포함 하 여 적절 하 게 작동에 필요한 Oracle 서버 제품 및 해당 클라이언트 구성 요소에 대 한 패치를 서비스 (IIS) 구성 요소 서비스 (또는 Microsoft Transaction Server, Windows NT를 사용 하는 경우) 등입니다.  
  
> [!NOTE]  
>  다음 지침을 정확 하지 않을 완전히 Oracle FTP 사이트 사용자가 변경 될 수 있습니다.  
  
### <a name="to-download-the-oracle-software-patches"></a>Oracle 소프트웨어 패치를 다운로드 하려면  
  
1.  Oracle ftp.oracle.com에서 공용 FTP 사이트에 연결 합니다. 사용자 ID가 "익명"이 고 암호는 전자 메일 주소입니다.  
  
2.  다음 디렉터리로 이동 합니다. /server/wgt_tech/server/windowsNT 합니다.  
  
3.  Windows 95, Windows 98 및 Windows NT/Windows 2000에 대 한 가장 관련성이 높은 패치를 다운로드 하려면 Oracle-7.3 또는 8.0의 버전에 대 한 하위 디렉터리로 이동 합니다. 두 개의 하위 디렉터리는 /73patchsets 및 /80patchsets입니다.  
  
4.  두 SQL, Oracle의 네트워크 기술에 대 한 패치를 다운로드 하려면 * Net 또는 Net8를 다음 디렉터리로 이동: 네트워크입니다.  
  
 웹 브라우저에서이 FTP 사이트에 액세스 작동 하지 않을 수 있습니다. 문제가 발생 하는 경우 "기존" FTP 클라이언트를 사용 하거나 DOS 명령 프롬프트를 사용 하십시오.  
  
> [!NOTE]  
>  Oracle 현재 버전의 버그 수정 및 다음 소프트웨어 패치를 사용 하 여 이전 버전에 해당 retrofits, 때문에 사용할 수 있는 최신 패치를 다운로드 하는 것이 좋습니다. Oracle Server 클라이언트 구성 요소에 대 한 경우 특히 그렇습니다. 이러한 패치를 설치 하는 방법에 대 한 질문이 있으면 Oracle 지원에 문의 합니다.
