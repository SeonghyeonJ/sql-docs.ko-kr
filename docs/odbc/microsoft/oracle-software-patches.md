# <a name="oracle-software-patches"></a>Oracle 소프트웨어 패치
> [!IMPORTANT]  
>  이 기능은 이후 버전의 Windows에서 제거 될 예정입니다. 새 개발 작업에서는 이 기능을 사용하지 않도록 하고, 현재 이 기능을 사용하는 애플리케이션은 수정하세요. 대신 Oracle에서 제공 하는 ODBC 드라이버를 사용 합니다.  
  
 Oracle server 제품 및 클라이언트 구성 요소에 대 한 패치는 Oracle 용 Microsoft ODBC 드라이버, Microsoft OLE DB Provider for Oracle, 인터넷 정보 서비스 (IIS), 구성 요소 서비스 (또는 Windows NT를 사용 하는 경우) 등 여러 Microsoft 제품 및 기술에서 적절 한 기능을 수행 하는 데 필요 합니다.  
  
> [!NOTE]  
>  Oracle FTP 사이트가 변경 될 수 있으므로 다음 지침이 완전히 정확 하지 않을 수 있습니다.  
  
### <a name="to-download-the-oracle-software-patches"></a>Oracle 소프트웨어 패치를 다운로드 하려면  
  
1.  Oracle-ftp.oracle.com에서 공용 FTP 사이트에 연결 합니다. 사용자 ID는 "anonymous"이 고 암호는 전자 메일 주소입니다.  
  
2.  다음 디렉터리로 이동 합니다./server/wgt_tech/server/windowsNT.  
  
3.  Windows 95, Windows 98 및 Windows NT/Windows 2000에 가장 적합 한 패치를 다운로드 하려면 해당 버전의 Oracle-7.3 또는 8.0에 대 한 하위 디렉터리로 이동 합니다. 두 하위 디렉터리는/73patchsets 및/80patchsets.입니다.  
  
4.  Oracle의 네트워크 기술 (SQL * Net 또는 Net8)에 대 한 패치를 다운로드 하려면 다음 디렉터리로 이동 합니다./network.  
  
 웹 브라우저에서 FTP 사이트에 액세스 하지 못할 수 있습니다. 문제가 발생 하는 경우 "기존" FTP 클라이언트를 사용 하거나 DOS 명령 프롬프트를 사용 하십시오.  
  
> [!NOTE]  
>  Oracle은 현재 버전의 버그를 수정 하 고 소프트웨어 패치를 사용 하 여 이전 버전으로 retrofits 때문에 사용 가능한 최신 패치를 다운로드 하는 것이 좋습니다. 이는 Oracle Server 클라이언트 구성 요소에서 특히 그렇습니다. 이러한 패치를 설치 하는 방법에 대 한 질문이 있는 경우 Oracle 지원에 문의 하세요.
