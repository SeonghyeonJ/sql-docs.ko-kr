# <a name="oracle-software-patches"></a>Oracle 소프트웨어 패치
> [!IMPORTANT]  
>  이 기능은 이후 버전의 Windows에서 제거됩니다. 새 개발 작업에서는 이 기능을 사용하지 않도록 하고, 현재 이 기능을 사용하는 애플리케이션은 수정하세요. 대신 오라클에서 제공하는 ODBC 드라이버를 사용합니다.  
  
 오라클 서버 제품 및 클라이언트 구성 요소에 대한 패치는 오라클용 Microsoft ODBC 드라이버, 오라클용 Microsoft OLE DB 공급자, 인터넷 정보 서비스(IIS), 구성 요소 서비스(또는 Windows NT를 사용하는 경우 Microsoft 트랜잭션 서버) 등 여러 Microsoft 제품 및 기술의 적절한 작동을 위해 필요합니다.  
  
> [!NOTE]  
>  오라클 FTP 사이트가 변경될 수 있으므로 다음 지침이 완전히 정확하지 않을 수 있습니다.  
  
### <a name="to-download-the-oracle-software-patches"></a>Oracle 소프트웨어 패치를 다운로드하려면  
  
1.  oracle-ftp.oracle.com 공개 FTP 사이트에 연결합니다. 사용자 ID는 "익명"이며 암호는 전자 메일 주소입니다.  
  
2.  다음 디렉토리로 이동합니다: /서버/wgt_tech/서버/windowsNT.  
  
3.  Windows 95, Windows 98 및 Windows NT/Windows 2000과 가장 적합한 패치를 다운로드하려면 오라클 버전(7.3 또는 8.0)의 하위 디렉토리로 이동하십시오. 두 하위 디렉터리들은 /73patchset 및 /80patchset입니다.  
  
4.  오라클의 네트워크 기술(SQL*Net 또는 Net8)에 대한 패치를 다운로드하려면 다음 디렉터리인 /network로 이동하십시오.  
  
 웹 브라우저에서 이 FTP 사이트에 액세스하는 것이 작동하지 않을 수 있습니다. 문제가 발생하면 "기존" FTP 클라이언트를 사용하거나 DOS 명령 프롬프트를 사용하십시오.  
  
> [!NOTE]  
>  오라클은 현재 버전에서 버그를 수정한 다음 소프트웨어 패치를 사용하여 이전 버전으로 수정하므로 사용 가능한 최신 패치를 다운로드하는 것이 좋습니다. 이는 오라클 서버 클라이언트 구성 요소의 경우 특히 그렇습니다. 이러한 패치 설치에 대한 질문이 있는 경우 Oracle 지원에 문의하십시오.
