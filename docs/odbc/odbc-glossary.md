# <a name="odbc-glossary"></a>ODBC 용어 설명
## <a name="a"></a>변수를 잠그기 위한  
 **액세스 계획**  
 SQL 문을 실행 하려면 데이터베이스 엔진에 의해 생성 된 계획 합니다. 3. 같은 3 세대 언어에서 컴파일된 실행 코드와 동일  
  
 **집계 함수**  
 많이 사용 되는 값의 그룹에서 단일 값을 생성 하는 함수 **GROUP BY** 하 고 **HAVING** 절. 집계 함수는 **AVG**, **개수**를 **최대**, **MIN**, 및 **합계**합니다. 라고도 *집합 함수*합니다. *참고* 스칼라 함수입니다.  
  
 **ANSI**  
 American National Standards Institute 합니다. ODBC API는 ANSI 호출 수준 인터페이스를 기반으로 합니다.  
  
 **APD**  
 *참조* (APD) 응용 프로그램 매개 변수 설명자입니다.  
  
 **API**  
 응용 프로그램 프로그래밍 인터페이스입니다. 응용 프로그램 요청 하위 수준 서비스 하 고 수행 하는 데 사용 하는 루틴 집합이 있습니다. ODBC 함수는 ODBC API로 구성 됩니다.  
  
 **application**  
 ODBC api에서 함수를 호출 하는 실행 프로그램입니다.  
  
 **응용 프로그램 매개 변수 설명자 (APD)**  
 응용 프로그램에서 지정 된 모든 변환 하기 전에 SQL 문에서 사용 된 동적 매개 변수를 설명 하는 설명자입니다.  
  
 **응용 프로그램 행 설명자 ()**  
 열 메타 데이터 및 응용 프로그램에서 지정 된 모든 데이터 변환을 수행 하는 데이터 행을 설명 하는 응용 프로그램의 버퍼에서 데이터를 나타내는 설명자입니다.  
  
 **ARD**  
 *참조* 응용 프로그램 행 설명자 ().  
  
 **자동 커밋 모드**  
 트랜잭션이 커밋되는 실행 된 후에 즉시 트랜잭션 커밋 모드입니다.  
  
## <a name="b"></a>B  
 **동작 변경**  
 ODBC 3에서 특정 기능 변경 *.x* 동작을 ODBC 2. *x* 동작 또는 그 반대의 경우도 마찬가지입니다. SQL_ATTR_ODBC_VERSION 환경 특성을 변경 하 여 발생 합니다.  
  
 **이진 대형 개체 (BLOB)**  
 특정 개수의 같은 255 바이트를 통해 모든 이진 데이터입니다. 일반적으로 훨씬 더 오래 걸립니다. 일반적으로 이러한 데이터 전송 되 고 부분에서 데이터 원본에서 검색 합니다. 라고도 *긴 데이터*입니다.  
  
 **binding**  
 동사는 응용 프로그램 변수를 사용 하 여 결과 집합의 열 또는 SQL 문에서 매개 변수를 연결 하는 작업입니다. 으로 명사로 연결 합니다.  
  
 **바인딩 오프셋**  
 모든 바인딩된 열 또는 매개 변수를 생성 하는 데이터를 새 주소에 대 한 데이터 버퍼 주소 및 길이/표시기 버퍼 주소를 추가 하는 값입니다.  
  
 **block cursor**  
 한 번에 둘 이상의 데이터 행을 인출 수 있는 커서입니다.  
  
 **buffer**  
 부분 응용 프로그램 및 드라이버 간에 데이터를 전달 하는 데 사용 되는 응용 프로그램 메모리입니다. 버퍼 쌍에서 경우가:는 *데이터 버퍼* 와 *데이터 길이가 버퍼*합니다.  
  
 **byte**  
 8 비트 또는 하나의 8 진수입니다. *참고* 8 진수입니다.  
  
## <a name="c"></a>C  
 **C 데이터 형식**  
 이 경우 응용 프로그램에서에서 C 프로그램에서 변수의 데이터 형식입니다.  
  
 **catalog**  
 데이터베이스의 모양을 설명 하는 데이터베이스의 시스템 테이블의 집합입니다. 라고도 *스키마* 하거나 *데이터 사전*합니다.  
  
 **카탈로그 함수**  
 데이터베이스의 카탈로그에서 정보를 검색 하는 데 사용 되는 ODBC 함수입니다.  
  
 **CLI**  
 *See* API.  
  
 **client/server**  
 하나 이상의 클라이언트가 서버를 통해 데이터에 액세스 하는 데이터베이스 액세스 전략입니다. 클라이언트는 일반적으로 서버 컨트롤 데이터베이스 액세스 하는 동안 사용자 인터페이스를 구현 합니다.  
  
 **column**  
 정보 행의 단일 항목에 대 한 컨테이너입니다. 라고도 *필드*합니다.  
  
 **commit**  
 변경 내용을 트랜잭션에 영구적으로 유지 되도록 합니다.  
  
 **동시성**  
 동시에 동일한 데이터에 액세스 하려면 둘 이상의 트랜잭션의 수 있습니다.  
  
 **적합성 수준**  
 불연속 집합 드라이버 또는 데이터 원본에서 지 원하는 기능입니다. ODBC는 API 적합성 수준 및 SQL 적합성 수준을 정의합니다.  
  
 **연결**  
 드라이버 및 데이터 원본의 특정 인스턴스.  
  
 **연결 검색**  
 데이터 원본에 연결할 네트워크를 검색 합니다. 연결 검색 몇 가지 단계를 포함할 수 있습니다. 예를 들어, 사용자 먼저 네트워크에서 서버를 찾아 하 고 데이터베이스를 특정 서버를 찾아볼 수 있습니다.  
  
 **연결 핸들**  
 연결에 대 한 정보를 포함 하는 데이터 구조에 대 한 핸들입니다.  
  
 **현재 행**  
 현재 커서가 가리키는 행입니다. 위치 지정된 작업을 현재 행에서 작동합니다.  
  
 **cursor**  
 부분 응용 프로그램에 데이터 행을 반환 하는 소프트웨어입니다. 아마도 깜박이 커서 터미널; 컴퓨터의 이름을 딴 해당 커서 화면에 현재 위치를 나타내는, 처럼 결과 집합에서 커서 결과 집합의 현재 위치를 나타냅니다.  
  
## <a name="d"></a>d  
 **data buffer**  
 데이터를 전달 하는 데 사용 되는 버퍼입니다. 연결 버퍼 된 데이터를 *데이터 길이가 버퍼*합니다.  
  
 **데이터 사전**  
 *참조* 카탈로그입니다.  
  
 **데이터 길이가 버퍼**  
 해당 값의 길이 전달 하는 데 사용 하는 버퍼 *데이터 버퍼*합니다. 데이터 값이 null로 끝나는 같은 표시기를 저장할 데이터 길이가 버퍼도 사용 됩니다.  
  
 **데이터 원본**  
 데이터 사용자가 액세스 하 고 연관 된 운영 체제, DBMS 및 네트워크 플랫폼 (있는 경우)입니다.  
  
 **데이터 형식**  
 형식 데이터의 일부입니다. ODBC C 및 SQL 데이터 형식을 정의합니다. *참고* 형식 표시기입니다.  
  
 **data-at-execution column**  
 후 데이터는 전송 하는 열 **SQLSetPos** 라고 합니다. 데이터 행 집합 버퍼에 배치 하는 것이 아니라 실행 시간에 전송 되기 때문에 그렇게 명명 된 합니다. Long 데이터 실행 시 부분에서 일반적으로 전송 됩니다.  
  
 **data-at-execution parameter**  
 매개 변수는 데이터가 후 보내집니다 **SQLExecute** 또는 **SQLExecDirect** 라고 합니다. 이라는 매개 변수 버퍼에 배치 하는 대신 SQL 문이 실행 될 때 전송 됩니다. Long 데이터 실행 시 부분에서 일반적으로 전송 됩니다.  
  
 **database**  
 DBMS의 데이터를 불연속 컬렉션입니다. DBMS도 합니다.  
  
 **데이터베이스 엔진**  
 DBMS 및 구문 분석 하 고, SQL 문을 실행 하 고, 실제 데이터에 액세스 하는 소프트웨어입니다.  
  
 **DBMS**  
 데이터베이스 관리 시스템입니다. 물리적 데이터베이스와 사용자 간의 소프트웨어 계층입니다. DBMS는 데이터베이스에 대한 모든 액세스를 관리합니다.  
  
 **DBMS 기반 드라이버**  
 독립 실행형 데이터베이스 엔진을 통해 실제 데이터에 액세스 하는 드라이버입니다.  
  
 **DDL**  
 데이터 정의 언어입니다. 데이터를 조작 하이 아니라 정의 SQL 문입니다. 예를 들어 **CREATE TABLE**를 **CREATE INDEX**를 **부여**, 및 **해지**합니다.  
  
 **delimited identifier**  
 특수 문자를 포함 하거나 키워드 (따옴표 붙은 식별자 라고도 함)와 일치 수 있도록 따옴표 식별자에에서 포함 된 식별자입니다.  
  
 **descriptor**  
 동적 매개 변수 또는 열 데이터에 대 한 정보를 포함 하는 데이터 구조입니다. 설명자의 물리적으로 나타낸 정의 되어 있지 않습니다. 설명자 핸들을 사용 하 여 ODBC 함수를 호출 하 여 해당 필드를 조작에 의해서만 설명자에 대 한 직접 액세스를 확보 하는 응용 프로그램입니다.  
  
 **데스크톱 데이터베이스**  
 DBMS는 개인용 컴퓨터에서 실행 하도록 설계 되었습니다. 일반적으로 이러한 Dbms 독립 실행형 데이터베이스 엔진을 제공 하지 않으며 파일 기반 드라이버를 통해 액세스 해야 합니다. 일반적으로 이러한 드라이버의 엔진 트랜잭션과 SQL에 대 한 지원을 줄었습니다. 예를 들어, dBASE, Paradox, Btrieve, 또는 Microsoft FoxPro?  
  
 **diagnostic**  
 특정 핸들을 사용 하는 마지막 함수는 호출에 대 한 진단 정보를 포함 하는 레코드입니다. 진단 레코드 환경, 연결, 문 및 설명자 핸들을 사용 하 여 연결 됩니다.  
  
 **DML**  
 데이터 조작 언어입니다. 과 반대로 데이터를 정의 조작 하는 SQL 문입니다. 예를 들어 **삽입**, **업데이트**합니다 **삭제**, 및 **선택**합니다.  
  
 **driver**  
 ODBC API 함수를 노출 하는 라이브러리 루틴입니다. 드라이버는 단일 DBMS와 관련이 있습니다.  
  
 **드라이버 관리자**  
 응용 프로그램에 대 한 드라이버에 대 한 액세스를 관리 하는 라이브러리 루틴입니다. 드라이버 관리자를 로드 하 고 언로드합니다 (또는 연결할 및에서 연결을 끊습니다) 드라이버 및 전달 ODBC 함수 호출을 올바른 드라이버입니다.  
  
 **드라이버 설치 DLL**  
 드라이버 관련 설치 및 구성 기능을 포함 하는 DLL입니다.  
  
 **dynamic cursor**  
 행을 검색할 수 있으므로 스크롤 가능 커서를 업데이트, 삭제 또는 결과 집합에 삽입 합니다.  
  
 **dynamic SQL**  
 Embedded sql는 SQL 문을 생성 되 고 런타임에 컴파일되는 형식입니다. *참고* 정적 SQL입니다.  
  
## <a name="e"></a>E  
 **Embedded SQL**  
 COBOL 또는 ODBC 3.와 같은 다른 언어로 작성 된 프로그램에 직접 포함 된 SQL 문이 포함 된 SQL을 사용 하지 않습니다. *참고 항목* 정적 SQL *고* 동적 SQL입니다.  
  
 **environment**  
 데이터에 액세스 하는 전역 컨텍스트 환경과 관련 된는 전역 환경에서 모든 연결 목록과 같은 특성에는 모든 정보입니다.  
  
 **환경 핸들**  
 환경에 대 한 정보를 포함 하는 데이터 구조에 대 한 핸들입니다.  
  
 **escape 절**  
 SQL 문 절입니다.  
  
 **execute**  
 SQL 문을 실행 합니다.  
  
## <a name="f"></a>F  
 **fat cursor**  
 *참조* 커서를 차단 합니다.  
  
 **fetch**  
 결과 집합에서 하나 이상의 행을 검색 합니다.  
  
 **field**  
 *참조* 열입니다.  
  
 **파일 기반 드라이버**  
 실제 데이터를 직접 액세스 하는 드라이버입니다. 이 경우 드라이버 포함 데이터베이스 엔진 및 드라이버와 데이터 원본으로 작동 합니다.  
  
 **파일 데이터 원본**  
 데이터 원본 연결에 대 한 정보는.dsn 파일에 저장 됩니다.  
  
 **외래 키**  
 열 또는 다른 테이블의 기본 키와 일치 하는 테이블의 열입니다.  
  
 **정방향 전용 커서**  
 결과 집합 및 일반적으로 앞으로 이동할 수 있는 커서를 한 번에 하나의 행을 인출 합니다. 대부분의 관계형 데이터베이스에는 정방향 전용 커서에만 지원합니다.  
  
## <a name="h"></a>H  
 **handle**  
 고유 하 게 파일 또는 데이터 구조와 같은 무엇 인가 식별 하는 값입니다. 핸들은 및 사용 하 여 만들지만 작업을 식별 하는 다른 소프트웨어에 의해 전달 되는 소프트웨어에만 의미가 있습니다. ODBC 환경, 연결, 문 및 설명자에 대 한 핸들을 정의합니다.  
  
## <a name="i"></a>I  
 **IPD (구현 매개 변수 설명자)**  
 응용 프로그램에서 지정 된 모든 변환 후 SQL 문에서 사용 된 동적 매개 변수를 설명 하는 설명자입니다.  
  
 **구현 행 설명자 (IRD)**  
 응용 프로그램에서 지정 된 모든 변환 하기 전에 데이터 행을 설명 하는 설명자입니다.  
  
 **설치 관리자 DLL**  
 ODBC 구성 요소를 설치 하 고 데이터 원본을 구성 하는 DLL입니다.  
  
 **무결성 향상 기능**  
 SQL의 하위 집합을 데이터베이스의 무결성을 유지 하도록 설계 되었습니다.  
  
 **인터페이스 적합성 수준**  
 드라이버;에서 지 원하는 ODBC 3.7 인터페이스의 수준 Core, 수준 1 또는 수준 2를 수 있습니다.  
  
 **interoperability**  
 다양 한 Dbms의 데이터에 액세스할 때 동일한 코드를 사용 하 여 응용 프로그램의 기능입니다.  
  
 **IPD**  
 *참조* IPD (구현 매개 변수 설명자).  
  
 **IRD**  
 *참조* 구현 행 설명자 (IRD).  
  
 **ISO/IEC**  
 조직/International Electrotechnical Commission 국제 표준 합니다. ODBC API는 ISO/IEC 호출 수준 인터페이스를 기반으로 합니다.  
  
## <a name="j"></a>J  
 **join**  
 지정 된 열을 두 개 이상의 테이블의 행에서 값을 비교 하 여 연결 하는 관계형 데이터베이스에서 작업 합니다.  
  
## <a name="k"></a>K  
 **key**  
 열 또는 열 값을 가진 행을 식별 합니다. *참고 항목* 외래 키 *고* 기본 키입니다.  
  
 **keyset**  
 집합 행을 다시 인출 혼합 또는 키 집합 커서에서 사용 되는 키입니다.  
  
 **keyset-driven cursor**  
 키 집합 커서를 사용 하 여 업데이트 및 삭제 된 행을 검색 하는 스크롤 가능한 커서입니다.  
  
## <a name="l"></a>L  
 **literal**  
 SQL 문에서 실제 데이터 값의 문자 표현입니다.  
  
 **locking**  
 사용 되는 DBMS 행 다중 사용자 환경에 액세스를 제한 하는 프로세스입니다. DBMS 행 또는 행을 나타내는 행이 포함 된 실제 페이지에서 다소 일반적으로 설정 하거나 페이지 잠겨 합니다.  
  
 **long 데이터**  
 특정 길이 255 바이트 또는 문자 등을 통해 모든 이진 또는 문자 데이터입니다. 일반적으로 훨씬 더 오래 걸립니다. 일반적으로 이러한 데이터 전송 되 고 부분에서 데이터 원본에서 검색 합니다. 라고도 *BLOB*s 또는 *CLOB*s입니다.  
  
## <a name="m"></a>M  
 **컴퓨터 데이터 원본**  
 데이터 원본 연결에 대 한 정보 (예를 들어, 레지스트리) 시스템에 저장 됩니다.  
  
 **수동 커밋 모드**  
 트랜잭션을 명시적으로 호출 하 여 커밋해야 때는 트랜잭션 커밋 모드 **SQLTransact**합니다.  
  
 **메타 데이터**  
 SQL 문에서 매개 변수 또는 결과 집합의 열을 설명 하는 데이터입니다. 예를 들어, 데이터 형식, 바이트 길이 매개 변수의 전체 자릿수입니다.  
  
 **다중 계층 드라이버**  
 *참조* DBMS 기반 드라이버입니다.  
  
## <a name="n"></a>N  
 **NULL 값**  
 명시적으로 할당 된 값이 없는 것입니다. 특히 NULL 값을 0 또는 빈에서 다릅니다.  
  
## <a name="o"></a>O  
 **octet**  
 8 비트 또는 1 바이트입니다. *참고* 바이트입니다.  
  
 **octet length**  
 포함 된 데이터 버퍼의 8 진수 길이입니다.  
  
 **ODBC**  
 데이터베이스 연결을 엽니다. 응용 프로그램 데이터 원본의 데이터에에서 액세스할 수 있습니다 하는 루틴의 표준 집합을 정의 하는 API에 대 한 사양입니다.  
  
 **ODBC 관리자**  
 설치 관리자 데이터 원본을 구성 하는 DLL을 호출 하는 실행 프로그램입니다.  
  
 그룹 열기  
 표준을 게시 하는 회사입니다. 특히 SQL 액세스 그룹 (강하) 표준에 게시합니다.  
  
 **optimistic concurrency**  
 행을 잠그지 동시성을 향상 하는 전략입니다. 대신, 업데이트 되거나 삭제 될 때, 전에 커서 경우 변경 되었는지 마지막으로 읽은 이후에를 확인 합니다. 그렇다면, update 또는 delete 실패 합니다. *참고* 비관적 동시성입니다.  
  
 **외부 조인**  
 일치 및 일치 하지 않는 행을 반환 하는 조인 합니다. 일치 하지 않는 일치 하지 않는 행 테이블에서 모든 열의 값은 NULL로 설정 됩니다.  
  
 **owner**  
 테이블의 소유자입니다.  
  
## <a name="p"></a>P  
 **parameter**  
 매개 변수 표식 또는 물음표 (?)로 표시 하는 SQL 문에 변수입니다. 매개 변수는 응용 프로그램 변수와 문이 실행 될 때 검색 된 해당 값에 바인딩됩니다.  
  
 **매개 변수 설명자**  
 하거나 응용 프로그램 (응용 프로그램 매개 변수 설명자를 또는 APD)에서 또는 응용 프로그램 (구현에서 지정 된 모든 변환 후 지정 된 변환 하기 전에 SQL 문에서 사용 되는 런타임 매개 변수를 설명 하는 설명자 매개 변수 설명자 또는 IPD)입니다.  
  
 **작업 매개 변수 배열**  
 배열에 해당 매개 변수는 무시 해야 나타내기 위해 응용 프로그램을 설정할 수 있는 값을 포함 하는 **SQLExecDirect** 또는 **SQLExecute** 작업 합니다.  
  
 **매개 변수 상태 배열**  
 호출한 후에 매개 변수의 상태를 포함 하는 배열을 **SQLExecDirect** 하거나 **SQLExecute**합니다.  
  
 **pessimistic concurrency**  
 다른 트랜잭션에서 변경할 수 있도록 행을 잠그는 순차성을 구현 하기 위한 전략을 합니다. *참고 항목* 낙관적 동시성 *고* 순차성 합니다.  
  
 **위치 지정된 작업**  
 현재 행에서 사용 되는 모든 작업입니다. 예를 들어 업데이트를 배치 하 고 delete 문이 **SQLGetData**, 및 **SQLSetPos**합니다.  
  
 **위치 지정된 update 문**  
 현재 행에서 값을 업데이트 하는 데 사용 되는 SQL 문입니다.  
  
 **위치 지정된 delete 문**  
 현재 행을 삭제 하는 데 사용 되는 SQL 문입니다.  
  
 **prepare**  
 SQL 문을 컴파일할 합니다. SQL 문을 준비 하 여 액세스 계획을 생성 됩니다.  
  
 **기본 키**  
 열 또는 테이블의 행을 고유 하 게 식별 하는 열입니다.  
  
 **procedure**  
 하나 이상의 그룹에는 데이터베이스의 명명 된 개체로 저장 되는 SQL 문을 미리 컴파일된입니다.  
  
 **프로시저 열**  
 프로시저 호출 프로시저나 프로시저에서 만든 결과 집합의 열에서 반환한 값에서에서 사용 되는 인수입니다.  
  
## <a name="q"></a>Q  
 **qualifier**  
 하나 이상의 테이블을 포함 하는 데이터베이스입니다.  
  
 **query**  
 SQL 문입니다. 경우에 따라 의미로 사용을 **선택** 문입니다.  
  
 **따옴표 붙은 식별자**  
 특수 문자를 포함 하거나 (구분 기호로 분리 된 식별자로 SQL-92에 라고도) 키워드와 일치 수 있도록 따옴표 식별자에에서 포함 된 식별자입니다.  
  
## <a name="r"></a>R  
 **radix**  
 숫자 체계의 밑수입니다. 일반적으로 2 또는 10입니다.  
  
 **record**  
 *참조* 행입니다.  
  
 **결과 집합**  
 실행 하 여 생성 된 행 집합을 **선택** 문입니다.  
  
 **return code**  
 ODBC 함수에서 반환 된 값입니다.  
  
 **롤백**  
 원래 상태로 트랜잭션에 의해 변경 된 값을 반환 합니다.  
  
 **row**  
 특정 엔터티를 설명 하는 관련된 열 집합이 있습니다. 라고도 *레코드*합니다.  
  
 **행 설명자**  
 일 (구현 행 설명자를 또는 IRD) 응용 프로그램 또는 응용 프로그램 (응용 프로그램 행 설명자를 또는) 지정 된 모든 변환 후 지정 된 변환 하기 전에 결과 집합의 열을 설명 하는 설명자입니다.  
  
 **행 작업 배열**  
 배열에서 해당 행을 무시 해야 함을 나타내기 위해 응용 프로그램을 설정할 수 있는 값을 포함 하는 **SQLSetPos** 작업 합니다.  
  
 **행 상태 배열**  
 호출한 후에 행의 상태를 포함 하는 배열 **SQLFetch**하십시오 **SQLFetchScroll**, 또는 **SQLSetPos**합니다.  
  
 **rowset**  
 블록 커서에서 단일 인출에서 반환 된 행 집합이 있습니다.  
  
 **버퍼 행 집합**  
 버퍼에서 전체 행 집합에 대 한 데이터에서 반환 되는 결과 집합의 열에 바인딩됩니다.  
  
## <a name="s"></a>S  
 **SAG**  
 *참조* SQL 액세스 그룹 (강하).  
  
 **스칼라 함수**  
 단일 값에서 단일 값을 생성 하는 함수입니다. 예를 들어, 문자 데이터의 대/소문자를 변경 하는 함수입니다.  
  
 **schema**  
 *참조* 카탈로그입니다.  
  
 **스크롤 가능 커서**  
 결과 집합을 통해 앞으로 또는 뒤로 이동할 수 있는 커서입니다.  
  
 **serializability**  
 같은지 여부를 두 트랜잭션이 동시에 실행 결과 생성 하는 이러한 트랜잭션을 직렬 (또는 순차) 실행 하는 것입니다. 직렬화 가능 트랜잭션 데이터베이스 무결성을 유지 해야 합니다.  
  
 **서버 데이터베이스**  
 DBMS는 클라이언트/서버 환경에서 실행 되도록 설계 되었습니다. 이러한 Dbms는 SQL 및 트랜잭션에 대 한 풍부한 지원을 제공 하는 독립 실행형 데이터베이스 엔진을 제공 합니다. DBMS 기반 드라이버를 통해 액세스 됩니다. 예를 들어, Oracle, Informix, DB/2 또는 Microsoft® SQL Server입니다.  
  
 **set 함수**  
 *참조* 집계 함수입니다.  
  
 **설치 DLL**  
 *참조* 드라이버 설치 DLL *고* 변환기 설치 DLL입니다.  
  
 **단일 계층 드라이버**  
 *참조* 파일 기반 드라이버입니다.  
  
 **SQL**  
 구조적된 쿼리 언어입니다. 관계형 데이터베이스에서 쿼리를 사용 하는 언어를 업데이트 하 고 데이터를 관리 합니다.  
  
 **(강하)의 SQL 액세스 그룹**  
 관련된 SQL Dbms를 사용 하 여 회사의 업계 컨소시엄 합니다. Open Group의 호출 수준 인터페이스의 SQL 액세스 그룹에서 원래 수행 된 작업을 기반으로 합니다.  
  
 **SQL 적합성 수준**  
 드라이버;에서 지 원하는 SQL-92 문법의 수준 항목, FIPS 전환, 중간 또는 전체를 수 있습니다.  
  
 **SQL 데이터 형식**  
 데이터 형식의 열 또는 해당 매개 변수에 데이터 원본에 저장 됩니다.  
  
 **SQLSTATE**  
 특정 오류를 나타내는 5 자로 이루어진 값입니다.  
  
 **SQL 문**  
 키워드로 시작 하 고 수행할 작업을 완벽 하 게 설명 하는 SQL의 전체 구입니다. 예를 들어 선택 *에서 주문 합니다. SQL 문의 문이 혼동 해서는 안됩니다.  
  
 **state**  
 항목의 잘 정의 된 조건입니다. 예를 들어, 연결에 할당 되지 않은, 할당, 연결 및 이므로 데이터를 포함 하 여 7 상태입니다. 항목 특정 상태에 있을 경우에 특정 작업을 수행할 수 있습니다. 예를 들어, 경우에 할당 된 상태에서 및 not, 예를 들어, 연결된 된 상태에 있을 때 연결을 해제할 수 있습니다.  
  
 **상태 전환**  
 다른 하나의 상태에서 항목 이동 합니다. ODBC 환경, 연결 및 문 엄격한 상태 전환을 정의합니다.  
  
 **statement**  
 SQL 문에 관련 된 모든 정보에 대 한 컨테이너입니다. 문은 SQL 문이 혼동 해서는 안됩니다.  
  
 **statement handle**  
 문에 대 한 정보를 포함 하는 데이터 구조에 대 한 핸들입니다.  
  
 **static cursor**  
 업데이트를 검색할 수 없는 스크롤 가능 커서를 삭제 하거나 결과 집합에 삽입 합니다. 일반적으로 결과 집합의 복사본을 만들어 구현 합니다.  
  
 **static SQL**  
 포함 된 sql 문 SQL에서 하드 코드 및 컴파일된 프로그램의 나머지 컴파일될 때 사용 되는 형식입니다. *참고* 동적 SQL입니다.  
  
 **저장된 프로시저**  
 *참조* 프로시저입니다.  
  
## <a name="t"></a>T  
 **table**  
 행의 컬렉션입니다.  
  
 **thunking**  
 16 비트를 변환 하는 16 비트 응용 프로그램이 32 비트 ODBC 드라이버와 함께 사용 하는 경우 32 비트 주소에 또는 그 반대의 경우를 해결 합니다.  
  
 **transaction**  
 원자 단위 작업입니다. 전체 트랜잭션에서 작업을 완료 해야 합니다. 트랜잭션의 일부가 실패 하면 전체 트랜잭션이 실패 합니다.  
  
 **트랜잭션 격리**  
 다른 모든 트랜잭션의 영향을 받지 않도록에서 한 트랜잭션 격리는 작업.  
  
 **트랜잭션 격리 수준**  
 측정값을 얼마나 잘 트랜잭션의 격리 됩니다. 트랜잭션 격리 수준은 5 가지가 있습니다. 커밋되지 않은 읽기에서는 커밋된 및 반복 가능한 읽기, 직렬화 및 버전 관리를 읽습니다.  
  
 **translator DLL**  
 DLL에서 하나의 문자 집합을 다른 데이터를 변환 하는 데 사용 합니다.  
  
 **변환기 설치 DLL**  
 Translator 별 설치 및 구성 기능을 포함 하는 DLL입니다.  
  
 **2 단계 커밋**  
 프로세스 두 단계에서 분산된 트랜잭션을 커밋하는 중입니다. 첫 번째 단계에서 트랜잭션 프로세서 트랜잭션의 모든 부분이 커밋될 수 있음을 확인 합니다. 두 번째 단계에서는 모든 부분 트랜잭션이 커밋됩니다. 트랜잭션의 일부를 나타내는 경우 첫 번째 단계에서는 커밋할 수 없는 두 번째 단계는 발생 하지 않습니다. ODBC 2 단계 커밋 지원 하지 않습니다.  
  
 **유형 표시기**  
 정수 값에 전달 하거나 응용 프로그램 변수, 매개 변수, 또는 열 데이터 형식을 지정 하기 위해 ODBC 함수에서 반환 합니다. ODBC C와 SQL 데이터 형식에 대 한 유형 표시기를 정의합니다.  
  
## <a name="v"></a>V  
 **보기**  
 하나 이상의 테이블에서 데이터를 볼 수 있는 대안 일반적으로 보기를가 하나 이상의 테이블에서 열의 하위 집합으로 생성 됩니다. Odbc에서 뷰는 테이블에 일반적으로 동일 합니다.
