# <a name="odbc-glossary"></a>ODBC 용어 설명
## <a name="a"></a>A  
 **액세스 계획**  
 SQL 문을 실행 하기 위해 데이터베이스 엔진에 의해 생성 된 계획입니다. C와 같은 3세대 언어에서 컴파일된 실행 코드와 동일합니다.  
  
 **집계 함수**  
 **GROUP BY** 및 **HAVING** 절과 함께 자주 사용되는 값 그룹에서 단일 값을 생성하는 함수입니다. 집계 함수에는 **AVG,** **COUNT,** **MAX,** **MIN**및 **SUM이**포함됩니다. *세트 함수라고도 합니다.* *스칼라 기능도 참조하십시오.*  
  
 **Ansi**  
 미국 국립 표준 연구소. ODBC API는 ANSI 통화 수준 인터페이스를 기반으로 합니다.  
  
 **APD**  
 응용 프로그램 매개 변수 설명자(APD)를 *참조하십시오.*  
  
 **API**  
 응용 프로그램 프로그래밍 인터페이스. 응용 프로그램이 하위 수준 서비스를 요청하고 수행하는 데 사용하는 루틴 집합입니다. ODBC API는 ODBC 함수로 구성됩니다.  
  
 **애플리케이션**  
 ODBC API에서 함수를 호출하는 실행 가능한 프로그램입니다.  
  
 **응용 프로그램 매개 변수 설명자(APD)**  
 응용 프로그램에서 지정한 변환 전에 SQL 문에 사용되는 동적 매개 변수를 설명하는 설명자입니다.  
  
 **응용 프로그램 행 설명자(ARD)**  
 응용 프로그램 버퍼의 열 메타데이터 및 데이터를 나타내는 설명자로, 응용 프로그램에서 지정한 데이터 변환 다음에 따라 데이터 행을 설명합니다.  
  
 **Ard**  
 응용 프로그램 행 설명자(ARD)를 *참조하십시오.*  
  
 **자동 커밋 모드**  
 트랜잭션이 실행된 직후커밋되는 트랜잭션 커밋 모드입니다.  
  
## <a name="b"></a>b  
 **동작 변경**  
 ODBC *3.x* 동작에서 ODBC *2.x* 동작으로 특정 기능이 변경된 경우 또는 그 반대의 경우도 마찬가지입니다. SQL_ATTR_ODBC_VERSION 환경 특성을 변경하여 발생합니다.  
  
 **이진 큰 개체(BLOB)**  
 255와 같은 특정 바이트 수에 대한 모든 이진 데이터입니다. 일반적으로 훨씬 더 길다. 이러한 데이터는 일반적으로 부분적으로 데이터 원본으로 전송되고 검색됩니다. *긴 데이터라고도*합니다.  
  
 **바인딩**  
 동사로, 결과 집합에 열을 연결 하는 행위 또는 응용 프로그램 변수와 SQL 문의 매개 변수입니다. 명사, 협회.  
  
 **바인딩 오프셋**  
 데이터 버퍼 주소에 추가된 값으로, 모든 바운드 열 또는 매개 변수 데이터에 대한 길이/표시기 버퍼 주소가 새 주소를 생성합니다.  
  
 **블록 커서(block cursor)**  
 한 번에 두 개 이상의 데이터 행을 가져올 수 있는 커서입니다.  
  
 **버퍼**  
 응용 프로그램과 드라이버 간에 데이터를 전달하는 데 사용되는 응용 프로그램 메모리입니다. 버퍼는 종종 쌍으로 제공됩니다: *데이터 버퍼* 및 *데이터 길이 버퍼*.  
  
 **바이트**  
 8 비트 또는 하나의 옥텟. *옥텟도 참조하십시오.*  
  
## <a name="c"></a>C  
 **C 데이터 형식**  
 C 프로그램에서 변수의 데이터 형식(이 경우 응용 프로그램)입니다.  
  
 **카탈로그**  
 데이터베이스의 모양을 설명하는 데이터베이스의 시스템 테이블 집합입니다. *스키마* 또는 *데이터 사전이라고도*합니다.  
  
 **카탈로그 기능**  
 데이터베이스 카탈로그에서 정보를 검색하는 데 사용되는 ODBC 함수입니다.  
  
 **CLI**  
 *참조* Api.  
  
 **클라이언트/서버**  
 하나 이상의 클라이언트가 서버를 통해 데이터에 액세스하는 데이터베이스 액세스 전략입니다. 서버가 데이터베이스 액세스를 제어하는 동안 클라이언트는 일반적으로 사용자 인터페이스를 구현합니다.  
  
 **열**  
 행의 단일 정보 항목에 대한 컨테이너입니다. *필드라고도*합니다.  
  
 **커밋**  
 트랜잭션을 영구적으로 변경합니다.  
  
 **동시성**  
 동시에 동일한 데이터에 액세스하는 두 개 이상의 트랜잭션의 기능입니다.  
  
 **적합성 수준**  
 드라이버 또는 데이터 원본에서 지원하는 별도의 기능 집합입니다. ODBC는 API 준수 수준 및 SQL 준수 수준을 정의합니다.  
  
 **연결**  
 드라이버 및 데이터 원본의 특정 인스턴스입니다.  
  
 **연결 브라우징**  
 연결할 데이터 원본에 대한 네트워크 검색 연결 검색에는 여러 단계가 포함될 수 있습니다. 예를 들어 사용자는 먼저 네트워크를 찾아 서버를 찾아본 다음 특정 서버에서 데이터베이스를 찾아볼 수 있습니다.  
  
 **연결 핸들**  
 연결에 대한 정보가 포함된 데이터 구조에 대한 핸들입니다.  
  
 **현재 행**  
 현재 커서가 가리키는 행입니다. 위치 가있는 작업은 현재 행에 작용합니다.  
  
 **커서**  
 응용 프로그램에 데이터 행을 반환하는 소프트웨어입니다. 아마도 컴퓨터 단말기에서 깜박이는 커서의 이름을 따서 명명되었습니다. 커서가 화면의 현재 위치를 나타내는 것처럼 결과 집합의 커서는 결과 집합의 현재 위치를 나타냅니다.  
  
## <a name="d"></a>D  
 **데이터 버퍼**  
 데이터를 전달하는 데 사용되는 버퍼입니다. 데이터 버퍼와 연관된 경우가 많으며 *데이터 길이 버퍼입니다.*  
  
 **데이터 사전(data dictionary)**  
 *카탈로그를 참조하십시오.*  
  
 **데이터 길이 버퍼**  
 해당 *데이터 버퍼에서*값의 길이를 전달하는 데 사용되는 버퍼입니다. 데이터 길이 버퍼는 데이터 값이 null-종료되는지 여부와 같은 표시기를 저장하는 데도 사용됩니다.  
  
 **데이터 소스**  
 사용자가 액세스하려는 데이터와 관련 운영 체제, DBMS 및 네트워크 플랫폼(있는 경우)입니다.  
  
 **데이터 형식**  
 데이터 조각의 유형입니다. ODBC는 C 및 SQL 데이터 형식을 정의합니다. 또한 유형 표시기를 *참조하십시오.*  
  
 **실행 시 데이터 열**  
 **SQLSetPos가** 호출된 후 데이터가 전송되는 열입니다. 따라서 데이터가 행 집합 버퍼에 배치되지 않고 실행 시 전송되기 때문에 명명됩니다. 긴 데이터는 일반적으로 실행 시 부분적으로 전송됩니다.  
  
 **실행 시 데이터 매개 변수**  
 **SQLExecute** 또는 **SQLExecDirect** 후에 데이터가 전송되는 매개 변수가 호출됩니다. 따라서 매개 변수 버퍼에 배치되지 않고 SQL 문이 실행될 때 데이터가 전송되기 때문에 명명됩니다. 긴 데이터는 일반적으로 실행 시 부분적으로 전송됩니다.  
  
 **데이터베이스**  
 DBMS의 개별 데이터 컬렉션입니다. 또한 DBMS.  
  
 **데이터베이스 엔진**  
 SQL 문을 구문 분석 하고 실행 하 고 물리적 데이터에 액세스 하는 DBMS의 소프트웨어입니다.  
  
 **Dbms**  
 데이터베이스 관리 시스템. 물리적 데이터베이스와 사용자 간의 소프트웨어 계층입니다. DBMS는 데이터베이스에 대한 모든 액세스를 관리합니다.  
  
 **DBMS 기반 드라이버**  
 독립 실행형 데이터베이스 엔진을 통해 물리적 데이터에 액세스하는 드라이버입니다.  
  
 **DDL**  
 데이터 정의 언어입니다. 데이터를 조작하는 것이 아니라 정의하는 SQL의 명령문입니다. 예를 들어 **테이블 만들기,** **인덱스 만들기,** **GRANT**및 **해지**.  
  
 **구분 식별자(delimited identifier)**  
 식별자 따옴표 문자로 둘러싸인 식별자로 특수 문자 또는 일치 키워드(따옴표로 인용된 식별자라고도 함)를 포함할 수 있습니다.  
  
 **설명자**  
 열 데이터 또는 동적 매개 변수에 대한 정보를 포함하는 데이터 구조입니다. 설명자의 물리적 표현은 정의되지 않습니다. 응용 프로그램은 설명자 핸들을 사용하여 ODBC 함수를 호출하여 해당 필드를 조작해야만 설명자에 직접 액세스할 수 있습니다.  
  
 **데스크톱 데이터베이스**  
 개인용 컴퓨터에서 실행되도록 설계된 DBMS입니다. 일반적으로 이러한 DBMS는 독립 실행형 데이터베이스 엔진을 제공하지 않으며 파일 기반 드라이버를 통해 액세스해야 합니다. 이러한 드라이버의 엔진은 일반적으로 SQL 및 트랜잭션에 대한 지원을 줄입니다. 예를 들어, dBASE, 역설, 브리브, 또는 마이크로 소프트® FoxPro®.  
  
 **진단**  
 특정 핸들을 사용한 마지막 함수에 대한 진단 정보가 포함된 레코드입니다. 진단 레코드는 환경, 연결, 명령문 및 설명자 핸들과 연결됩니다.  
  
 **DML**  
 데이터 조작 언어. 데이터를 정의하는 것이 아니라 조작하는 SQL의 명령문입니다. 예를 들어 **삽입**, **업데이트**, **삭제**및 **선택**.  
  
 **드라이버**  
 ODBC API의 함수를 노출하는 루틴 라이브러리입니다. 드라이버는 단일 DBMS에만 해당됩니다.  
  
 **드라이버 관리자**  
 응용 프로그램의 드라이버에 대한 액세스를 관리하는 루틴 라이브러리입니다. 드라이버 관리자는 드라이버를 로드하고 언로드(또는 연결 및 연결 해제)하고 ODBC 함수에 대한 호출을 올바른 드라이버로 전달합니다.  
  
 **드라이버 설정 DLL**  
 드라이버별 설치 및 구성 기능이 포함된 DLL입니다.  
  
 **동적 커서(dynamic cursor)**  
 결과 집합에 업데이트, 삭제 또는 삽입된 행을 검색할 수 있는 스크롤 가능한 커서입니다.  
  
 **동적 SQL**  
 SQL 문이 생성되고 런타임에 컴파일되는 임베디드 SQL 유형입니다. 정적 *SQL도 참조하십시오.*  
  
## <a name="e"></a>E  
 **임베디드 SQL**  
 코볼 이나 C. ODBC와 같은 다른 언어로 작성 된 프로그램에 직접 포함 된 SQL 문은 포함 된 SQL을 사용 하지 않습니다. 정적 SQL *및* 동적 *SQL도 참조하십시오.*  
  
 **환경**  
 데이터에 액세스하는 전역 컨텍스트; 환경과 연관된 정보는 해당 환경의 모든 연결 목록과 같이 본질적으로 전역적인 모든 정보입니다.  
  
 **환경 핸들**  
 환경에 대한 정보가 포함된 데이터 구조에 대한 핸들입니다.  
  
 **이스케이프 절**  
 SQL 문의 절입니다.  
  
 **실행**  
 SQL 문을 실행합니다.  
  
## <a name="f"></a>F  
 **뚱뚱한 커서**  
 블록 커서를 *참조하십시오.*  
  
 **가져올**  
 결과 집합에서 하나 이상의 행을 검색합니다.  
  
 **필드(field)**  
 열을 *참조하십시오.*  
  
 **파일 기반 드라이버**  
 실제 데이터에 직접 액세스하는 드라이버입니다. 이 경우 드라이버는 데이터베이스 엔진을 포함하며 드라이버와 데이터 원본 모두의 역할을 합니다.  
  
 **파일 데이터 소스**  
 연결 정보가 .dsn 파일에 저장되는 데이터 원본입니다.  
  
 **외래 키(foreign key)**  
 다른 테이블의 기본 키와 일치하는 테이블의 열 또는 열입니다.  
  
 **정방향 전용 커서**  
 결과 집합을 통해 앞으로만 이동할 수 있으며 일반적으로 한 번에 하나의 행만 가져오는 커서입니다. 대부분의 관계형 데이터베이스는 정방향 전용 커서만 지원합니다.  
  
## <a name="h"></a>H  
 **처리**  
 파일 또는 데이터 구조와 같은 것을 고유하게 식별하는 값입니다. 핸들은 만들고 사용하는 소프트웨어에만 의미가 있지만 다른 소프트웨어에 의해 전달되어 사물을 식별합니다. ODBC는 환경, 연결, 명령문 및 설명자에 대한 핸들을 정의합니다.  
  
## <a name="i"></a>I  
 **IPD(구현 매개 변수 설명자)**  
 응용 프로그램에서 지정한 변환 후 SQL 문에 사용되는 동적 매개 변수를 설명하는 설명자입니다.  
  
 **구현 행 설명자(IRD)**  
 응용 프로그램에서 지정한 변환 전에 데이터 행을 설명하는 설명자입니다.  
  
 **설치 프로그램 DLL**  
 ODBC 구성 요소를 설치하고 데이터 원본을 구성하는 DLL입니다.  
  
 **무결성 향상 시설**  
 데이터베이스의 무결성을 유지하도록 설계된 SQL의 하위 집합입니다.  
  
 **인터페이스 적합성 수준**  
 드라이버가 지원하는 ODBC 3.7 인터페이스의 수준; 코어, 레벨 1 또는 레벨 2일 수 있습니다.  
  
 **상호 운용성**  
 한 응용 프로그램이 다른 DBMS의 데이터에 액세스할 때 동일한 코드를 사용할 수 있는 기능입니다.  
  
 **IPD**  
 *참조* IPD(구현 매개 변수 설명자).  
  
 **Ird**  
 *참조* 구현 행 설명자(IRD).  
  
 **ISO/IEC**  
 국제 표준 기구/국제 전기 기술 위원회. ODBC API는 ISO/IEC 통화 수준 인터페이스를 기반으로 합니다.  
  
## <a name="j"></a>J  
 **가입**  
 지정된 열의 값을 일치시켜 두 개 이상의 테이블의 행을 연결하는 관계형 데이터베이스의 작업입니다.  
  
## <a name="k"></a>K  
 **키**  
 값이 행을 식별하는 열 또는 열입니다. 외래 키 *및* 기본 *키도 참조하십시오.*  
  
 **키**  
 혼합 또는 키 집합 기반 커서에서 행을 다시 가져오기 위해 사용하는 키 집합입니다.  
  
 **키 집합 커서(keyset-driven cursor)**  
 키 집합을 사용하여 업데이트되고 삭제된 행을 검색하는 스크롤 가능한 커서입니다.  
  
## <a name="l"></a>L  
 **리터럴**  
 SQL 문에서 실제 데이터 값의 문자 표현입니다.  
  
 **잠금**  
 DBMS가 다중 사용자 환경에서 행에 대한 액세스를 제한하는 프로세스입니다. DBMS는 일반적으로 행 또는 행 또는 페이지가 잠겨 있음을 나타내는 행을 포함하는 실제 페이지에 비트를 설정합니다.  
  
 **긴 데이터**  
 255바이트 또는 문자와 같은 특정 길이의 이진 또는 문자 데이터입니다. 일반적으로 훨씬 더 길다. 이러한 데이터는 일반적으로 부분적으로 데이터 원본으로 전송되고 검색됩니다. *BLOB*또는 *CLOB라고도*합니다.  
  
## <a name="m"></a>M  
 **컴퓨터 데이터 소스**  
 연결 정보가 시스템에 저장되는 데이터 원본(예: 레지스트리)입니다.  
  
 **수동 커밋 모드**  
 **SQLTransact**를 호출하여 트랜잭션을 명시적으로 커밋해야 하는 트랜잭션 커밋 모드입니다.  
  
 **메타 데이터**  
 SQL 문의 매개 변수 또는 결과 집합의 열을 설명하는 데이터입니다. 예를 들어 데이터 형식, 바이트 길이 및 매개 변수의 정밀도입니다.  
  
 **다중 계층 드라이버**  
 *참조* DBMS 기반 드라이버.  
  
## <a name="n"></a>N  
 **NULL 값**  
 명시적으로 할당된 값이 없습니다. 특히 NULL 값은 0 또는 공백과 다릅니다.  
  
## <a name="o"></a>O  
 **옥텟**  
 8비트 또는 1바이트. *바이트도 참조하십시오.*  
  
 **옥텟 길이**  
 버퍼 또는 버퍼에 포함된 데이터의 옥텟 길이입니다.  
  
 **ODBC**  
 데이터베이스 연결을 엽니다. 응용 프로그램이 데이터 원본의 데이터에 액세스할 수 있는 표준 루틴 집합을 정의하는 API에 대한 사양입니다.  
  
 **ODBC 관리자**  
 설치 관리자 DLL을 호출하여 데이터 원본을 구성하는 실행 가능한 프로그램입니다.  
  
 그룹 열기  
 표준을 게시하는 회사입니다. 특히 SQL 액세스 그룹(SAG) 표준을 게시합니다.  
  
 **낙관적 동시성**  
 행이 잠기지 않는 동시성을 높이기 위한 전략입니다. 대신 업데이트하거나 삭제하기 전에 커서가 마지막으로 읽은 이후 변경되었는지 확인합니다. 이 경우 업데이트 또는 삭제가 실패합니다. *비관적 동시성도 참조하십시오.*  
  
 **외부 조인**  
 일치하는 행과 일치하지 않는 행이 모두 반환되는 조인입니다. 일치하지 않는 행의 일치하지 않는 테이블의 모든 열 값은 NULL로 설정됩니다.  
  
 **소유자**  
 테이블의 소유자입니다.  
  
## <a name="p"></a>P  
 **매개 변수**  
 매개 변수 마커 또는 물음표(?)로 표시된 SQL 문의 변수입니다. 매개 변수는 응용 프로그램 변수에 바인딩되고 문이 실행될 때 해당 값이 검색됩니다.  
  
 **매개 변수 설명자**  
 응용 프로그램에서 지정한 변환 전(응용 프로그램 매개 변수 설명자 또는 APD) 또는 응용 프로그램에서 지정한 변환 후(구현 매개 변수 설명자 또는 IPD) SQL 문에 사용되는 런타임 매개 변수를 설명하는 설명자입니다.  
  
 **매개 변수 작업 배열**  
 **SQLExecDirect** 또는 **SQLExecute** 작업에서 해당 매개 변수를 무시해야 함을 나타내기 위해 응용 프로그램이 설정할 수 있는 값을 포함하는 배열입니다.  
  
 **매개 변수 상태 배열**  
 **SQLExecDirect** 또는 **SQLExecute를**호출한 후 매개 변수의 상태를 포함하는 배열입니다.  
  
 **비관적 동시성(pessimistic concurrency)**  
 다른 트랜잭션이 변경할 수 없도록 행이 잠겨 있는 serializability를 구현하기 위한 전략입니다. 또한 낙관적 동시성 *및* 직렬화 가능성을 *참조하십시오.*  
  
 **위치 작업**  
 현재 행에 작용하는 모든 작업입니다. 예를 들어 위치 업데이트 및 삭제 문, **SQLGetData**및 **SQLSetPos**.  
  
 **위치 업데이트 문**  
 현재 행의 값을 업데이트하는 데 사용되는 SQL 문입니다.  
  
 **위치 삭제 문**  
 현재 행을 삭제하는 데 사용되는 SQL 문입니다.  
  
 **준비**  
 SQL 문을 컴파일합니다. 액세스 계획은 SQL 문을 준비하여 만들어집니다.  
  
 **기본 키(primary key)**  
 테이블의 행을 고유하게 식별하는 열 또는 열입니다.  
  
 **절차**  
 데이터베이스에 명명된 개체로 저장되는 하나 이상의 미리 컴파일된 SQL 문의 그룹입니다.  
  
 **프로시저 열**  
 프로시저 호출의 인수, 프로시저에서 반환되는 값 또는 프로시저에서 만든 결과 집합의 열입니다.  
  
## <a name="q"></a>Q  
 **한정자**  
 하나 이상의 테이블을 포함하는 데이터베이스입니다.  
  
 **쿼리**  
 SQL 문입니다. SELECT **문을** 의미하는 데 사용되는 경우도 있습니다.  
  
 **인용 식별자**  
 특수 문자 또는 일치 키워드(SQL-92에서 구분된 식별자로도 알려짐)를 포함할 수 있도록 식별자 따옴표 문자로 둘러싸인 식별자입니다.  
  
## <a name="r"></a>R  
 **근원**  
 숫자 시스템의 기본입니다. 보통 2 또는 10.  
  
 **레코드**  
 *행을 참조하십시오.*  
  
 **결과 집합**  
 **SELECT** 문을 실행하여 만든 행 집합입니다.  
  
 **코드 반환**  
 ODBC 함수에서 반환되는 값입니다.  
  
 **롤백(roll back)**  
 트랜잭션에 의해 변경된 값을 원래 상태로 반환합니다.  
  
 **행**  
 특정 엔터티를 설명하는 관련 열 집합입니다. *레코드라고도*합니다.  
  
 **행 설명자**  
 응용 프로그램에서 지정한 변환 전(구현 행 설명자 또는 IRD) 또는 응용 프로그램에서 지정한 변환 후(응용 프로그램 행 설명자 또는 ARD) 후에 결과 집합의 열을 설명하는 설명자입니다.  
  
 **행 작업 배열**  
 **SQLSetPos** 작업에서 해당 행을 무시해야 함을 나타내기 위해 응용 프로그램이 설정할 수 있는 값을 포함하는 배열입니다.  
  
 **행 상태 배열**  
 **SQLFetch,** **SQLFetchScroll**또는 **SQLSetPos를**호출한 후 행의 상태를 포함하는 배열입니다.  
  
 **행 집합**  
 블록 커서에 의해 단일 가져오기에서 반환된 행 집합입니다.  
  
 **행 집합 버퍼**  
 결과 집합의 열에 바인딩된 버퍼와 전체 행 집합에 대한 데이터가 반환됩니다.  
  
## <a name="s"></a>S  
 **Sag**  
 *참조* SQL 액세스 그룹(SAG)입니다.  
  
 **스칼라 기능**  
 단일 값에서 단일 값을 생성하는 함수입니다. 예를 들어 문자 데이터의 대/소문자를 변경하는 함수입니다.  
  
 **schema**  
 *카탈로그를 참조하십시오.*  
  
 **스크롤 가능한 커서**  
 결과 집합을 통해 앞으로 또는 뒤로 이동할 수 있는 커서입니다.  
  
 **순차성**  
 두 트랜잭션이 동시에 실행되는지 여부는 해당 트랜잭션의 직렬(또는 순차적) 실행과 동일한 결과를 생성합니다. 데이터베이스 무결성을 유지하려면 직렬화 가능한 트랜잭션이 필요합니다.  
  
 **서버 데이터베이스**  
 클라이언트/서버 환경에서 실행되도록 설계된 DBMS입니다. 이러한 DBMS는 SQL 및 트랜잭션에 대한 풍부한 지원을 제공하는 독립 실행형 데이터베이스 엔진을 제공합니다. DBMS 기반 드라이버를 통해 액세스됩니다. 예를 들어 오라클, 인포믹스, DB/2 또는 마이크로소프트® SQL Server.  
  
 **설정 기능**  
 집계 함수를 *참조하십시오.*  
  
 **설정 DLL**  
 드라이버 설정 DLL *및* 번역기 설정 DLL을 *참조하십시오.*  
  
 **단일 계층 드라이버**  
 파일 기반 드라이버를 *참조하십시오.*  
  
 **SQL**  
 구조화 된 쿼리 언어입니다. 관계형 데이터베이스에서 데이터를 쿼리, 업데이트 및 관리하는 데 사용하는 언어입니다.  
  
 **SQL 액세스 그룹(SAG)**  
 SQL DBMS와 관련된 기업의 산업 컨소시엄. 오픈 그룹의 통화 수준 인터페이스는 SQL 액세스 그룹에서 원래 수행한 작업을 기반으로 합니다.  
  
 **SQL 적합성 수준**  
 드라이버가 지원하는 SQL-92 문법 수준; 항목, FIPS 전환, 중급 또는 전체 일 수 있습니다.  
  
 **SQL 데이터 형식**  
 데이터 원본에 저장되는 열 또는 매개 변수의 데이터 형식입니다.  
  
 **SQLSTATE**  
 특정 오류를 나타내는 5자 값입니다.  
  
 **SQL 문**  
 SQL의 완전한 구는 키워드로 시작하여 취할 작업을 완전히 설명합니다. 예를 들어 * 주문에서 선택합니다. SQL 문은 문과 혼동해서는 안 됩니다.  
  
 **상태**  
 항목의 잘 정의된 조건입니다. 예를 들어 연결에는 할당되지 않은 상태, 할당됨, 연결됨 및 필요한 데이터를 포함하여 7개의 상태가 있습니다. 특정 작업은 항목이 특정 상태에 있는 경우에만 수행할 수 있습니다. 예를 들어 연결이 할당된 상태일 때만 연결을 해제할 수 있으며 연결된 상태에 있을 때는 해제할 수 없습니다.  
  
 **상태 전환**  
 한 상태에서 다른 상태로 항목의 이동입니다. ODBC는 환경, 연결 및 문에 대한 엄격한 상태 전환을 정의합니다.  
  
 **문을**  
 SQL 문과 관련된 모든 정보에 대한 컨테이너입니다. 문은 SQL 문과 혼동해서는 안 됩니다.  
  
 **문 핸들**  
 명령문에 대한 정보가 포함된 데이터 구조에 대한 핸들입니다.  
  
 **정적 커서(static cursor)**  
 결과 집합에서 업데이트, 삭제 또는 삽입을 검색할 수 없는 스크롤 가능한 커서입니다. 일반적으로 결과 집합의 복사본을 만들어 구현합니다.  
  
 **정적 SQL**  
 프로그램의 나머지 부분을 컴파일할 때 SQL 문이 하드 코딩되고 컴파일되는 임베디드 SQL 유형입니다. *동적 SQL도 참조하십시오.*  
  
 **저장 절차**  
 절차를 *참조하십시오.*  
  
## <a name="t"></a>T  
 **테이블**  
 행의 컬렉션입니다.  
  
 **썽킹**  
 16비트 응용 프로그램을 32비트 ODBC 드라이버와 함께 사용할 때 16비트 주소를 32비트 주소로 변환하거나 그 반대로 변환합니다.  
  
 **트랜잭션**  
 작업의 원자 단위입니다. 한 트랜잭션의 작업은 전체를 완료해야 합니다. 트랜잭션의 일부가 실패하면 전체 트랜잭션이 실패합니다.  
  
 **트랜잭션 격리**  
 다른 모든 트랜잭션의 영향에서 한 트랜잭션을 격리 하는 행위입니다.  
  
 **트랜잭션 격리 수준**  
 트랜잭션이 얼마나 잘 격리되는지에 대한 척도입니다. 커밋되지 않은 읽기, 커밋됨 읽기, 반복 읽기, 직렬화 가능 및 버전 전환의 다섯 가지 트랜잭션 격리 수준이 있습니다.  
  
 **번역기 DLL**  
 한 문자 집합에서 다른 문자 집합으로 데이터를 변환하는 데 사용되는 DLL입니다.  
  
 **번역기 설정 DLL**  
 번역기별 설치 및 구성 기능이 포함된 DLL입니다.  
  
 **2단계 커밋**  
 분산 트랜잭션을 두 단계로 커밋하는 프로세스입니다. 첫 번째 단계에서 트랜잭션 프로세서는 트랜잭션의 모든 부분이 커밋될 수 있음을 확인합니다. 두 번째 단계에서는 트랜잭션의 모든 부분이 커밋됩니다. 트랜잭션의 일부가 첫 번째 단계에서 커밋할 수 없음을 나타내는 경우 두 번째 단계는 발생하지 않습니다. ODBC는 2단계 커밋을 지원하지 않습니다.  
  
 **유형 표시기**  
 응용 프로그램 변수, 매개 변수 또는 열의 데이터 형식을 나타내기 위해 ODBC 함수에서 전달되거나 반환되는 정수 값입니다. ODBC는 C 및 SQL 데이터 형식모두에 대한 형식 표시기를 정의합니다.  
  
## <a name="v"></a>V  
 **보기**  
 하나 이상의 테이블에서 데이터를 보는 다른 방법입니다. 뷰는 일반적으로 하나 이상의 테이블에서 열의 하위 집합으로 만들어집니다. ODBC에서 뷰는 일반적으로 테이블과 동일합니다.
