# <a name="what39s-new-in-odbc-38"></a>ODBC 3.8의 새로운&#39;
윈도우 8 ODBC 3.8의 업데이트 된 버전을 포함. Windows 8의 ODBC 3.8에는 다음과 같은 기능이 포함되어 있습니다.  
  
-   [드라이버 인식 연결 풀링](../../odbc/reference/develop-app/driver-aware-connection-pooling.md)  
  
-   [비동기 실행(알림 방법)](../../odbc/reference/develop-app/asynchronous-execution-notification-method.md)  
  
-   [데이터 액세스 추적 (윈도우 8)](https://msdn.microsoft.com/library/windows/desktop/hh829624.aspx)  
  
-   명령줄에서 ODBC 데이터 원본, ODBC 드라이버, ODBC 성능 카운터 및 데이터 액세스 추적을 관리할 수 있도록 PowerShell 명령이 추가되었습니다.  자세한 내용은 [Windows 데이터 액세스 구성 요소 PowerShell 명령을](https://msdn.microsoft.com/library/windows/desktop/jj134064.aspx)참조하십시오.  
  
 윈도우 7ODBC, ODBC 3.8의 업데이트 된 버전이 포함되어 있습니다. ODBC 3.8에는 다음과 같은 기능이 포함되어 있습니다.  
  
-   연결 작업을 비동기적으로 실행합니다. 자세한 내용은 [비동기 실행(폴링 메서드)을](../../odbc/reference/develop-app/asynchronous-execution-polling-method.md)참조하십시오.  
  
-   스트리밍된 출력 매개 변수입니다. 자세한 내용은 [SQLGetData를 사용하여 출력 매개 변수 검색을](../../odbc/reference/develop-app/retrieving-output-parameters-using-sqlgetdata.md)참조하십시오.  
  
-   ODBC C 데이터 형식 확장성. 자세한 내용은 [ODBC의 C 데이터 유형을](../../odbc/reference/develop-app/c-data-types-in-odbc.md)참조하십시오.  
  
 드라이버 작성기는 [3.5 드라이버에서 3.8 드라이버로 업그레이드를](../../odbc/reference/develop-driver/upgrading-a-3-5-driver-to-a-3-8-driver.md)읽어야 합니다.  
  
 ODBC 3.8 드라이버가 있는 ODBC 3.x 및 ODBC 2.x 응용 프로그램에서 비동기 연결 작업을 사용할 수 있습니다.  
  
 자세한 내용은 [호환성 매트릭스](../../odbc/reference/develop-app/compatibility-matrix.md)를 참조하십시오.
