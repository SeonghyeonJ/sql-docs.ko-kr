# <a name="types-of-descriptors"></a>설명자 형식
설명자는 다음 중 하나를 설명하는 데 사용됩니다.  
  
-   0개 이상의 매개 변수 집합입니다. 매개 변수 설명자는 다음을 설명하는 데 사용할 수 있습니다.  
  
    -   응용 프로그램에서 설정한 입력 동적 인수 또는 SQL의 **CALL** 문을 실행한 후 출력 동적 인수를 포함하는 응용 프로그램 *매개 변수 버퍼입니다.*  
  
    -   *구현 매개 변수 버퍼*. 입력 동적 인수의 경우 응용 프로그램에서 지정할 수 있는 데이터 변환 후 응용 프로그램 매개 변수 버퍼와 동일한 인수를 포함 합니다. 출력 동적 인수의 경우 응용 프로그램에서 지정할 수 있는 데이터 변환 전에 반환된 인수를 포함합니다.  
  
     입력 동적 인수의 경우 응용 프로그램은 동적 매개 변수 마커가 포함된 SQL 문을 실행하기 전에 응용 프로그램 매개 변수 설명자에서 작동해야 합니다. 입력 및 출력 동적 인수 모두에 대해 응용 프로그램은 데이터 변환을 달성하기 위해 구현 매개 변수 설명자의 데이터 형식과 다른 데이터 형식을 지정할 수 있습니다.  
  
-   데이터베이스 데이터의 단일 행입니다. 행 설명자는 다음을 설명하는 데 사용할 수 있습니다.  
  
    -   데이터베이스의 행을 포함하는 *구현 행 버퍼입니다.* 이러한 버퍼에는 개념적으로 데이터베이스에 기록하거나 데이터베이스에서 읽은 데이터가 포함됩니다. 그러나 저장된 데이터베이스 데이터 형식은 지정되지 않습니다. 데이터베이스는 구현 버퍼의 양식에서 데이터에 대한 추가 변환을 수행할 수 있습니다.  
  
    -   응용 프로그램에서 지정할 수 있는 데이터 변환 에 따라 응용 프로그램에 제시된 데이터 행을 포함하는 응용 프로그램 *행 버퍼입니다.*  
  
     응용 프로그램은 데이터베이스의 열 데이터가 응용 프로그램 변수에 표시되어야 하는 모든 경우에 응용 프로그램 행 설명자에서 작동합니다. 열 데이터의 데이터 변환을 달성하기 위해 응용 프로그램은 구현 행 설명자의 데이터 형식과 다른 데이터 형식을 지정할 수 있습니다.  
  
 설명자 형식은 다음 표에 요약되어 있습니다.  
  
|버퍼 유형|행|동적 매개 변수|  
|-----------------|----------|------------------------|  
|**응용 프로그램 버퍼**|응용 프로그램 행 설명자(ARD)|응용 프로그램 매개 변수 설명자(APD)|  
|**구현 버퍼**|구현 행 설명자(IRD)|구현 매개 변수 설명자(IPD)|  
  
 매개 변수 또는 행 버퍼의 경우 응용 프로그램이 구현 및 응용 프로그램 설명자의 해당 레코드에 다른 데이터 형식을 지정하는 경우 드라이버는 설명자 사용 시 데이터 변환을 수행합니다. 예를 들어 숫자 및 datetime 값을 문자 문자열 형식으로 변환할 수 있습니다. 유효한 변환의 경우 [부록 D: 데이터 유형을](../../../odbc/reference/appendixes/appendix-d-data-types.md)참조하십시오.  
  
 설명자는 다른 역할을 수행할 수 있습니다. 다른 문은 응용 프로그램이 명시적으로 할당하는 모든 설명자도 공유할 수 있습니다. 한 명령문의 행 설명자는 다른 명령문의 매개 변수 설명자 역할을 할 수 있습니다.  
  
 설명자가 데이터베이스 작업에 아직 사용되지 않은 경우에도 지정된 설명자가 응용 프로그램 설명자인지 구현 설명자인지 항상 알려져 있습니다. 구현이 암시적으로 할당하는 설명자의 경우 구현은 문 핸들을 기준으로 미리 정의된 행을 기록합니다. **SQLAllocHandle을** 호출하여 응용 프로그램이 할당하는 모든 설명자는 응용 프로그램 설명자입니다.
