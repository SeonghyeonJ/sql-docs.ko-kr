# <a name="data-sources"></a>솔루션 탐색기
*데이터 원본은* 단순히 데이터 원본입니다. 파일, DBMS의 특정 데이터베이스 또는 라이브 데이터 피드 일 수 있습니다. 데이터는 프로그램과 동일한 컴퓨터 또는 네트워크의 다른 컴퓨터에 있을 수 있습니다. 예를 들어 데이터 원본은 운영 체제®에서 실행 되는 Oracle DBMS로, Novell® Netware에서 액세스할 수 있습니다. 게이트웨이를 통해 액세스 하는 IBM DB2 DBMS 서버 디렉터리에 있는 Xbase 파일의 컬렉션입니다. 또는 로컬 Microsoft® Access 데이터베이스 파일  
  
 데이터 원본의 목적은 드라이버 이름, 네트워크 주소, 네트워크 소프트웨어 등 데이터에 액세스 하는 데 필요한 모든 기술 정보를 단일 위치에 수집 하 고 사용자 로부터 숨기는 것입니다. 사용자는 급여, 재고 및 담당자를 포함 하는 목록을 확인 하 고, 목록에서 급여를 선택 하 고, 급여 데이터가 있는 위치 또는 응용 프로그램의 위치를 몰라도 응용 프로그램이 급여 데이터에 연결할 수 있습니다.  
  
 *데이터 원본* 이라는 용어는 비슷한 용어와 혼동 해서는 안 됩니다. 이 설명서에서 *DBMS* 또는 *데이터베이스* 는 데이터베이스 프로그램 또는 엔진을 참조 합니다. 개인용 컴퓨터에서 실행 되도록 설계 된 *데스크톱 데이터베이스* 와 클라이언트/서버 환경에서 실행 되도록 설계 되 고 독립 실행형 데이터베이스 엔진 및 리치 SQL 및 트랜잭션 지원으로 구성 된 *서버 데이터베이스* 간에는 더 많은 차이점이 있습니다. 또한 *데이터베이스* 는 디렉터리의 Xbase 파일 컬렉션 또는 SQL Server의 데이터베이스와 같은 특정 데이터 컬렉션을 참조 합니다. 일반적으로이 설명서의 다른 곳에서 사용 되는 용어 *카탈로그* 또는 이전 버전의 ODBC에서 용어 *한정자* 와 동일 합니다.
