# <a name="standard-database-access-architectures"></a>표준 데이터베이스 액세스 아키텍처
앞섹션에서 설명한 데이터베이스 액세스 구성 요소를 살펴보면 프로그래밍 인터페이스와 데이터 스트림 프로토콜이라는 두 가지 구성 요소가 표준화에 적합한 것으로 나타났습니다. 다른 두 구성 요소인 IPC 메커니즘과 네트워크 프로토콜은 너무 낮은 수준에 있을 뿐만 아니라 네트워크 및 운영 체제에 크게 의존합니다. 표준화의 가능성을 제공하는 세 번째 접근 방식인 게이트웨이도 있습니다.
