### <a name="float-and-real-data-types"></a>float 및 real 데이터 형식  
다음 표에서 허용 되는 형식 및 형식의 배포 열에 리터럴 값을 삽입 하는 것에 대 한 규칙 정의 **부동 소수점** 하거나 **실제**합니다. 데이터 변환 규칙은 SQL Server에서와 동일 합니다. 자세한 내용은 [데이터 형식 변환](../t-sql/data-types/data-type-conversion-database-engine.md) MSDN에 있습니다.  
  
|리터럴 형식|형식|  
|----------------|----------|  
|문자열 리터럴로 **정수** 형식|' nnnnnnnnnnnn'<br /><br />예: '321312313123'|  
|문자열 리터럴로 **10 진수** 형식|'nnnnnn.nnnnn'<br /><br />예: '123344.34455'|  
|문자열 리터럴로 **부동 소수점** 형식|'n.nnnnnE+nn'<br /><br />예: ' 3.12323E + 14'|  
|정수 리터럴|nnnnnnnnnnnn<br /><br />예: 321312313123|  
|10 진수 리터럴|nnnnnn.nnnnn<br /><br />예: 123344.34455|  
|지점 리터럴은 부동|n.nnnnnE+nn<br /><br />예: 3.12323E+14|  
  
### <a name="int-bigint-tinyint-smallint-data-types"></a>int, bigint, tinyint, smallint 데이터 형식  
다음 표에서 허용 되는 형식 및 형식의 배포 열에 리터럴 값을 삽입 하는 것에 대 한 규칙을 정의 **int**, **bigint**하십시오 **tinyint**, 또는 **smallint**합니다. 데이터 소스는 지정 된 데이터 형식에 대 한 허용 범위를 초과할 수 없습니다. 예를 들어 범위 **tinyint** 0에서 255 사이 이며 범위 **int** 는-2,147,483,648에서 2,147,483,647입니다.  
  
|리터럴 형식|형식|변환 규칙|  
|------------|------|----------------|
|문자열 리터럴로 **정수** 형식|' nnnnnnnnnnnnnn'<br /><br />예: '321312313123'| 없음 |  
|정수 리터럴|nnnnnnnnnnnnnn<br /><br />예: 321312313123| 없음|  
|10 진수 리터럴|nnnnnn.nnnnn<br /><br />예: 123344.34455|값을 소수점의 오른쪽이 잘립니다.|  
  
### <a name="money-and-smallmoney-data-types"></a>money 및 smallmoney 데이터 형식  
리터럴 값을 money는 선택적 소수점 및 통화 기호 접두사를 가진 숫자로 표시 됩니다. 데이터 소스는 지정 된 데이터 형식에 대 한 허용 범위를 초과할 수 없습니다. 범위에 대 한 예를 들어 **smallmoney** -214,748.3648은 214,748.3647의 범위를 **money** 922337203685, 477.5807-922,337,203,685,477.5808 됩니다. 다음 표에서 허용 되는 형식 및 형식의 배포 열에 리터럴 값을 삽입 하는 것에 대 한 규칙 정의 **money** 하거나 **smallmoney**합니다.  
  
|리터럴 형식|형식|변환 규칙|  
|----------------|----------|--------------------|  
|문자열 리터럴로 **정수** 형식|' nnnnnnnn'<br /><br />예: '123433'|소수점 누락 된 숫자 값이 삽입 될 때 0으로 설정 됩니다. 예를 들어, 리터럴 '12345' 12345.0000로 삽입 됩니다.|  
|문자열 리터럴로 **10 진수** 형식|'nnnnnn.nnnnn'<br /><br />예: '123344.34455'|소수점 자릿수 4를 초과 하는 경우 값을 가장 가까운 값으로 반올림 됩니다. 예를 들어 값 '123344.34455' 123344.3446로 삽입 됩니다.|  
|문자열 리터럴로 **money** 형식|'$nnnnnn.nnnn'<br /><br />예: ' $ 123456.7890'|선택적인 통화 기호 값이 삽입 되지 않습니다.<br /><br />소수점 자릿수 4를 초과 하는 경우 값을 가장 가까운 값으로 반올림 됩니다.|  
|정수 리터럴|nnnnnnnn<br /><br />예: 123433|소수점 누락 된 숫자 값이 삽입 될 때 0으로 설정 됩니다. 예를 들어, 리터럴 12345 12345.0000로 삽입 됩니다.|  
|10 진수 리터럴|nnnnnn.nnnnn<br /><br />예: 123344.34455|소수점 자릿수 4를 초과 하는 경우 값을 가장 가까운 값으로 반올림 됩니다. 예를 들어 값 123344.34455 123344.3446로 삽입 됩니다.|  
|Money 리터럴|$nnnnnn.nnnn<br /><br />예: $ 123456.7890|선택적인 통화 기호 값이 삽입 되지 않습니다.<br /><br />소수점 자릿수 4를 초과 하는 경우 값을 가장 가까운 값으로 반올림 됩니다.|

### <a name="char-varchar-nchar-and-nvarchar-data-types"></a>char, varchar, nchar 및 nvarchar 데이터 형식  
다음 표에서 허용 되는 형식 및 형식의 배포 열에 리터럴 값을 삽입 하는 것에 대 한 규칙을 정의 **char**, **varchar**하십시오 **nchar** 및 **nvarchar**합니다. 데이터 소스 길이 데이터 형식에 대해 지정 된 크기를 초과할 수 없습니다. 데이터 소스 길이 크기 보다 작으면 합니다 **char** 또는 **nchar** 데이터 형식으로 데이터는 데이터 형식 크기에 연결할 빈 공간을 사용 하 여 오른쪽에 패딩 됩니다.  
  
|리터럴 형식|형식|변환 규칙|  
|----------------|----------|--------------------|  
|문자열 리터럴|형식: ' 문자열 '<br /><br />예: ' abc'| 없음|  
|유니코드 문자열 리터럴|형식: N'character 문자열 '<br /><br />예: 갖게 '|  없음 |

|Money 리터럴|형식: $nnnnnn.nnnnn<br /><br />예: $ 123456.99|통화 기호 값이 삽입 되지 않습니다. 통화 기호를 삽입 하려면 값을 문자열 리터럴로 삽입 합니다. 이 형식의 일치 됩니다 합니다 **dwloader** 도구 모든 리터럴 문자열 리터럴로 취급 합니다.<br /><br />쉼표는 허용 되지 않습니다.<br /><br />소수점 자릿수 2를 초과 하는 경우 값을 가장 가까운 값으로 반올림 됩니다. 예를 들어 값 123.946789 123.95로 삽입 됩니다.<br /><br />Money 리터럴을 삽입할 CONVERT 함수를 사용 하는 경우 기본 스타일 0 (쉼표를 사용 하지 및 소수점 뒤 두 자리)만 허용 됩니다.|
