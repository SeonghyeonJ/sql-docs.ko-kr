# <a name="o2ss0038-hierarchical-query-not-converted-error"></a>O2SS0038: 계층적 쿼리가 변환 되지 않음 (오류)

이 문서에서는 Oracle 용 SSMA (SQL Server Migration Assistant)가 하위 쿼리에서 복잡 한 함수를 포함 하는 계층적 쿼리를 변환할 수 없는 이유에 대해 설명 합니다.

## <a name="background"></a>배경

계층적 쿼리는 계층적 모델 데이터를 처리 하는 SQL 쿼리 유형입니다. 각 계층 구조 쿼리에서는 절을 사용 `START WITH` 하 여 시작 조건을 지정 해야 합니다. 그러면 절에 지정 된 `CONNECT BY` 하나 이상의 조건을 사용 하 여 모든 종속 행으로 trickle 됩니다.

계층 쿼리를 변환 하는 경우 SSMA는 해당 솔루션을 재귀 쿼리로 에뮬레이트합니다.

> [!NOTE]
> 재귀 CTE는 문 내에서 재귀 쿼리를 실행 하는 데 필요한 코드를 크게 간소화할 수 있습니다. Oracle의 `START WITH` 절은 재귀 부분을 `SELECT`포함 하는 첫 번째 중첩 된 첫 번째 중첩 `UNION`된 (재귀 부분)이 됩니다 `SELECT`.

집계 함수 (예: `MAX` `MIN`, 등)가 계층적 쿼리에서 하위 쿼리의 일부로 사용 되는 경우 ssma는 계층 구조 쿼리를 해당 SQL Server 쿼리로 변환 하 고 변환 오류를 생성할 수 없습니다.

## <a name="example"></a>예제

계층적 쿼리에서 하위 쿼리의 일부로 사용 되는 집계 함수 ( `MAX` `MIN`예:, 등)가 있는 다음 예를 살펴보십시오.

```sql
CREATE OR REPLACE PROCEDURE Hierarchy_Test AS
BEGIN
    FOR c IN (
        SELECT
            p.id,
            p.NAME,
            p.parent_id,
            pp.podr_cod,
            pp.count_chel,
            pp.date_start,
            pp.date_finish,
            pp.parent_podr_id
        FROM
            podr p,
            (SELECT *
             FROM podr_param pp
             WHERE pp.ROWID =
                (SELECT SUBSTR(MAX(TO_CHAR(date_start, 'YYYYMMDD') || rowid), 9)
                 FROM podr_param
                 WHERE podr_id=pp.podr_id)) pp
        WHERE p.id = pp.podr_id
        START WITH p.id = 123
        CONNECT BY PRIOR p.id = pp.parent_podr_id)
    LOOP
        dbms_output.put_line(
            RPAD(c.id || ' ', 15)
            || RPAD(c.podr_cod || ' ', 15)
            || RPAD(c.NAME || ' ', 15)
            || RPAD(TO_CHAR(c.date_start, 'DD.MM.YYYY') || ' ', 15)
            || RPAD(TO_CHAR(c.date_finish, 'DD.MM.YYYY'), 15));
    END LOOP;
END;
```

SSMA에서 위의 코드를 변환 하려고 하면 다음과 같은 오류 메시지가 생성 됩니다.

> O2SS0038: 계층적 쿼리가 변환 되지 않았습니다.

## <a name="possible-remedies"></a>가능한 해결 방법

이 솔루션은 집계 함수를 포함 하는 중첩 된 하위 쿼리를 단순화 하는 것입니다.

이 오류를 해결 하기 위해 Oracle에서 새 함수를 만들고 중첩 된 쿼리에서이 함수를 호출 합니다. 함수는 원하는 값을 계산 하 여 중첩 된 쿼리에 전달 합니다. 위의 쿼리를 위한 샘플 함수는 다음과 같습니다.

```sql
CREATE OR REPLACE function max_val(date_start DATE)
RETURN VARCHAR
IS
    val VARCHAR(20);
BEGIN
    SELECT ROWID
    FROM podr_param
    WHERE date_start =
        (SELECT MAX(date_start) INTO val FROM podr_param);

    RETURN (val);
END;
```

그런 다음 중첩 쿼리의 수정 된 프로시저 내에서이 함수를 호출할 수 있습니다. 그러면 오류가 해결 됩니다. 최종 결과 `START WITH` 는 Oracle의 절을 첫 번째 중첩 `SELECT`된로 변환 하 고 `UNION` , 재귀 부분은 다른 `SELECT`것으로 변환 합니다.

다음은 간소화 된 절차의 예입니다.

```sql
CREATE OR REPLACE PROCEDURE Hierarchy_Test AS
BEGIN
    FOR c IN (
        SELECT
            p.id,
            p.NAME,
            p.parent_id,
            pp.podr_cod,
            pp.count_chel,
            pp.date_start,
            pp.date_finish,
            pp.parent_podr_id
        FROM
            podr p,
            (SELECT *
             FROM podr_param pp
             WHERE pp.ROWID = max_val(date_start)) pp
        WHERE p.id = pp.podr_id
        START WITH p.id = 123
        CONNECT BY PRIOR p.id = pp.parent_podr_id)
    LOOP
        dbms_output.put_line(
            RPAD(c.id || ' ', 15)
            || RPAD(c.podr_cod || ' ', 15)
            || RPAD(c.NAME || ' ', 15)
            || RPAD(TO_CHAR(c.date_start, 'DD.MM.YYYY') || ' ', 15)
            || RPAD(TO_CHAR(c.date_finish, 'DD.MM.YYYY'), 15));
    END LOOP;
END;
```

이제 SSMA를 사용 하 여 결과를 SQL Server 변환할 수 있습니다.
