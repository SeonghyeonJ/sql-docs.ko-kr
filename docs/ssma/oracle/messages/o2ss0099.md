# <a name="o2ss0099-rownum-in-order-by-clause-is-not-supported-error"></a>O2SS0099: ORDER BY 절의 ROWNUM은 지원 되지 않습니다 (오류).

이 문서에서는 Oracle의 SSMA (SQL Server Migration Assistant)가 Oracle `ROWNUM` in `ORDER BY` 절을 포함 하는 문을 변환할 수 없는 이유에 대해 설명 합니다.

## <a name="background"></a>배경

Oracle `ROWNUM` 의사 열는 테이블에서 행이 선택 된 순서를 나타내는 숫자를 반환 합니다. 선택한 첫 번째 행에는 `ROWNUM` 의 `1`가 있습니다. 두 번째 행에는 `ROWNUM` 의 `2`가 있습니다.

In 절을 변환할 때 SSMA는 대부분의 `ROW_NUMBER()` 경우 오류 없이 에뮬레이션 기술을 사용 합니다. `ORDER BY` `ROWNUM` 그러나 `ORDER BY` 절이 `UNION` 쿼리의 일부로 사용 되는 경우 ssma는 오류 메시지를 생성 합니다.

## <a name="example"></a>예제

다음 예제를 참조하세요.

```sql
SELECT city
FROM customers

UNION

SELECT city
FROM employees
ORDER BY ROWNUM;
```

SSMA에서 위의 코드를 변환 하려고 하면 다음과 같은 오류 메시지가 생성 됩니다.

> O2SS0099: ORDER BY 절의 ROWNUM은 지원 되지 않습니다.

## <a name="possible-remedies"></a>가능한 해결 방법

이 솔루션은 괄호를 사용 하 여 하위 쿼리를 만들어 식을 간소화 하는 것입니다. 하위 쿼리를 만드는 경우 `ROWNUM` 에 `ORDER BY` 는 `UNION` 절을 사용 하 여 더 간단한 식으로 사용 되는 절을 구분 합니다. 다음은 간소화 된 쿼리의 예입니다.

```sql
SELECT
    city
FROM
    (SELECT city FROM customers
     UNION
     SELECT city FROM employees)
ORDER BY
    ROWNUM;
```

* O2SS0081: SELECT 문에서 지원 되지 않는 별표 및 ROWNUM
* O2SS0412: GROUPING SETS를 사용 하는 ROWNUM을 변환할 수 없습니다.

대부분의 경우에는 쿼리가 수행 하려는 작업을 `ROW_NUMBER()` 고려 하 고 에뮬레이션, `TOP` 절을 사용 하도록 다시 작성 하거나 `WHERE`, `GROUP BY`또는 `ORDER BY` 절에 고유한 식을 사용 합니다.
