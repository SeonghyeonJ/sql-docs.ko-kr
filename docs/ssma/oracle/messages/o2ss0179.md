# <a name="o2ss0179-packaged-variable-constant-data-type-not-recognized-error"></a>O2SS0179: 패키지 변수 (상수) 데이터 형식을 인식할 수 없습니다 (오류).

이 문서에서는 해당 유형의 변수가 패키지에서 사용 되는 경우 Oracle 용 SQL Server Migration Assistant (예: SSMA)에서 일부 Oracle 데이터 유형 `MLSLABEL`(예:)을 변환 하지 않는 이유를 설명 합니다.

## <a name="background"></a>배경

`MLSLABEL` 데이터 형식은 신뢰할 수 있는 Oracle에서 사용 되며 운영 체제 레이블의 이진 형식을 저장 하는 데 사용 됩니다. 신뢰할 수 있는 Oracle은 레이블을 사용 하 여 정보에 대 한 액세스를 제어 합니다. `MLSLABEL` 열의 최대 너비는 255 바이트입니다. 운영 체제에서 유효한 모든 레이블은 `MLSLABEL` 열에 삽입할 수 있습니다. `MLSLABEL` 열에 레이블을 삽입 하면 신뢰할 수 있는 Oracle에서 암시적으로 데이터를 레이블의 이진 형식으로 변환 합니다.

패키지 변수를 저장 하기 위해 SSMA는 특수 함수 ( `ssma_oracle.set_pv_varchar`예:)를 사용 `ssma_oracle.db_storage` 하 여 테이블에서 값을 검색 합니다. SSMA가 패키지 변수를 변환 하는 방법에 대 한 자세한 내용은 [Oracle 패키지 변수 에뮬레이션](../emulate-package-variables.md) 문서를 참조 하세요.

Ssma가 지원 되지 않는 데이터 형식을 변환 하려고 할 때마다 오류가 생성 됩니다. SSMA는 SQL Server에서 해당 형식을 찾을 수 없기 때문입니다.

## <a name="example"></a>예제

SSMA가 다음 PL/SQL 블록을 변환 하려고 할 때

```sql
CREATE OR REPLACE PACKAGE MLSLABEL_PKG
IS
    label mlslabel;
END MLSLABEL_PKG;

CREATE OR REPLACE PACKAGE BODY MLSLABEL_PKG
IS
BEGIN
    label := null;
END MLSLABEL_PKG;
```

다음 오류를 생성 합니다.

> O2SS0179: 패키지 변수 (상수) 데이터 형식이 인식 되지 않습니다.

## <a name="possible-remedies"></a>가능한 해결 방법

지원 되지 않는 데이터 형식을 다음과 같이 지원 되는 다른 데이터 형식 (예 `VARCHAR`:)으로 바꾸는 것이 좋습니다.

```sql
CREATE OR REPLACE PACKAGE MLSLABEL_PKG
IS
    label Varchar(255);
END MLSLABEL_PKG;

CREATE OR REPLACE PACKAGE BODY MLSLABEL_PKG
IS
BEGIN
    label := null;
END MLSLABEL_PKG;
```
