# <a name="o2ss0334-bulk-collect-into-clause-not-converted-error"></a>O2SS0334: 대량 수집 INTO 절이 변환 되지 않습니다 (오류).

이 문서에서는 Oracle 용 SSMA (SQL Server Migration Assistant) `BULK COLLECT INTO` 절이 포함 된 문 (컬렉션 범주 쿼리)을 변환할 수 없는 몇 가지 이유를 설명 합니다.

## <a name="background"></a>배경

컬렉션은 요소의 순서가 지정 된 그룹입니다. 목록, 배열 및 기타 친숙 한 데이터 형식을 포함 하는 일반적인 개념입니다. 각 요소에는 컬렉션에서의 위치를 결정 하는 고유한 첨자가 있습니다. 절 `BULK COLLECT INTO` 은 쿼리 출력을 컬렉션에 바인딩하여 PL/SQL 및 sql 엔진 간의 통신을 감소 합니다.

SSMA는 Oracle 컬렉션의 마이그레이션에 대 한 경로를 제공 하지만 목록 수가 `BULK COLLECT INTO` 컬렉션 집합과 일치 하지 않는 경우 ssma는 변환을 수행 하 고 오류 메시지를 생성할 수 없습니다.

## <a name="example"></a>예제

두 `dept` 개의 열 (`deptno` 및 `dname`)이 있고 `SELECT` 문에서 별표를 사용 하는 테이블의 다음 예를 살펴보겠습니다.

```sql
DECLARE
    TYPE test_deptno IS TABLE OF number;
    var_deptno test_deptno;
BEGIN
    SELECT
        *
    BULK COLLECT INTO
        var_deptno
    FROM
        dept;
END;
```

SSMA에서 위의 코드를 변환 하려고 하면 다음과 같은 오류 메시지가 생성 됩니다.

> O2SS0334: 대량 수집 INTO 절이 변환 되지 않았습니다.

## <a name="possible-remedies"></a>가능한 해결 방법

이 솔루션은 절에 `SELECT` 나열 된 열 수를 절에 `BULK COLLECT INTO` 사용 된 변수 수와 정규화 하는 것입니다. 이 예에서 별표는 `dept` 테이블에서 두 개의 열로 확장 되지만 `BULK COLLECT INTO` 절에는 하나의 변수만 있습니다.

예를 성공적으로 마이그레이션하려면 `deptno` `dept` 테이블에서 열만 페치 하거나 요구 사항에 따라 절에서 `BULK COLLECT INTO` 다른 변수를 추가 합니다. 아래 예제에서는 절에 추가 변수를 `BULK COLLECT INTO` 추가 하 여 쿼리를 수정 하는 방법을 보여 줍니다.

```sql
DECLARE
    TYPE test_deptno IS TABLE OF number;
    TYPE test_dname IS TABLE OF varchar2(20);
    var_deptno test_deptno;
    var_dname test_dname;
BEGIN
    SELECT
        deptno,
        dname
    BULK COLLECT INTO
        var_deptno,
        var_dname
    FROM
        dept;
END;
```
